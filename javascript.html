🟢 Basics — Q1 to Q12

Core JavaScript fundamentals every developer must know

Q1 What is JavaScript? What are its key features? ▼

JavaScript is a lightweight, interpreted, single-threaded, dynamically-typed programming language that conforms to the ECMAScript specification. Originally built for browsers, it now also runs server-side via Node.js.

  • Dynamic typing — variables can hold any type at runtime
  • First-class functions — functions are values, can be passed around
  • Prototype-based OOP — inheritance via prototypes, not classical classes
  • Single-threaded + Event Loop — non-blocking async via callbacks, promises, async/await
  • Closures — functions remember their lexical scope
  • Ubiquitous — runs in every browser; Node.js for backend
Interview tip: JavaScript is single-threaded but non-blocking because the event loop offloads I/O to the browser/OS.
Q2 What are JavaScript data types? ▼

JavaScript has 8 data types — 7 primitives and 1 object type:

  • Primitives: number, string, boolean, undefined, null, symbol (ES6), bigint (ES2020)
  • Object: Arrays, functions, dates, maps, sets — all are objects
typeof 42           // "number"
typeof "hello"       // "string"
typeof true         // "boolean"
typeof undefined    // "undefined"
typeof null         // "object" ← famous bug!
typeof Symbol()     // "symbol"
typeof 9007199254740993n // "bigint"
typeof {}           // "object"
typeof []           // "object" (use Array.isArray())
typeof function(){} // "function"
Q3 What is the difference between var, let, and const? ▼
  • var: Function-scoped. Hoisted and initialized to undefined. Can be re-declared and re-assigned. Avoid in modern JS.
  • let: Block-scoped. Hoisted but NOT initialized (Temporal Dead Zone). Can be re-assigned, not re-declared in same scope.
  • const: Block-scoped. Must be initialized. Cannot be re-assigned. Objects/arrays declared with const are still mutable.
var x = 1; var x = 2;  // OK — re-declared
let y = 1; y = 2;       // OK — re-assigned
const z = {};
z.name = "Alice";       // OK — object mutation
z = {};                  // ❌ TypeError — reassignment
Use const by default. Switch to let only when you need to reassign. Never use var.
Q4 What is the difference between == and === in JavaScript? ▼
  • == (Loose equality): Compares values after type coercion. Converts operands to the same type first.
  • === (Strict equality): Compares value AND type. No coercion.
0   == ""     // true  (both coerce to 0 / false)
0   === ""    // false
null == undefined // true
null === undefined // false
"1" == 1     // true
"1" === 1    // false
false == 0   // true
Always use === unless you explicitly need type coercion (e.g., x == null to catch both null and undefined).
Q5 What is Hoisting in JavaScript? ▼

Hoisting is JavaScript's behavior of moving declarations (not initializations) to the top of their scope during the compilation phase.

// var — hoisted and initialized to undefined
console.log(x); // undefined (no error)
var x = 5;

// let/const — hoisted but NOT initialized (TDZ)
console.log(y); // ReferenceError: Cannot access before init
let y = 5;

// Function declarations — fully hoisted
sayHi(); // "Hi!" — works before declaration
function sayHi() { console.log("Hi!"); }

// Function expressions — NOT hoisted
greet(); // TypeError: greet is not a function
var greet = function() {};
Q6 What are truthy and falsy values in JavaScript? ▼

Falsy values — evaluate to false in a boolean context. There are exactly 8:

false, 0, -0, 0n, "", '', ``, null, undefined, NaN

Everything else is truthy — including "0", "false", [], {}.

if ([]) console.log("truthy"); // "truthy" — empty array is truthy!
if ("0") console.log("truthy"); // "truthy" — non-empty string
Boolean([])  // true
Boolean("") // false
Q7 What is NaN? How do you check for it? ▼

NaN (Not a Number) is a special numeric value that results from invalid math operations. It has a quirk: NaN !== NaN (it's not equal to itself).

parseInt("abc")   // NaN
0 / 0            // NaN
Math.sqrt(-1)    // NaN

NaN === NaN      // false  ← unique!

// Checking for NaN
isNaN("abc")    // true  (but coerces: isNaN("") = false!)
Number.isNaN("abc") // false (strict — only true for actual NaN)
Number.isNaN(NaN)    // true  ← preferred method
Always use Number.isNaN(), not the global isNaN(), which coerces its argument first.
Q8 What is the difference between null and undefined? ▼
  • undefined: A variable has been declared but not yet assigned a value. Also returned by functions with no return statement, and accessing missing object properties.
  • null: An explicit assignment representing "no value" or "empty." Must be deliberately set by the developer.
let x;           // undefined (auto)
let y = null;   // null (intentional)

typeof undefined // "undefined"
typeof null      // "object" (historical bug)

null == undefined  // true  (loose)
null === undefined // false (strict)
Q9 What is type coercion in JavaScript? ▼

Type coercion is JavaScript's automatic conversion of values from one type to another when operators or conditions expect a different type.

// Implicit coercion (automatic)
"5" + 3        // "53"  — number coerced to string
"5" - 3        // 2     — string coerced to number
true + true   // 2     — booleans coerced to numbers
[] + []        // ""    — arrays coerced to strings
[] + {}        // "[object Object]"
{} + []        // 0     (in console — {} = empty block)

// Explicit coercion (manual)
Number("42")   // 42
String(42)    // "42"
Boolean(0)    // false
Q10 What are JavaScript's comparison quirks with objects? ▼

Primitives are compared by value. Objects are compared by reference (memory address). Two objects with identical contents are NOT equal unless they are the same object.

// Primitives — value comparison
"hello" === "hello"  // true

// Objects — reference comparison
{} === {}              // false (different objects in memory)
[] === []              // false

const a = { x: 1 };
const b = a;          // same reference
a === b                // true
a.x = 99;
console.log(b.x)       // 99 — mutation visible via b

// Deep equality — use libraries or JSON trick
JSON.stringify(a) === JSON.stringify(b)
Q11 What is the difference between pass by value and pass by reference? ▼
  • Primitives are passed by value — a copy is made. The original is unaffected.
  • Objects and arrays are passed by reference — both the variable and the parameter point to the same object in memory.
// Primitive — pass by value
function double(n) { n *= 2; }
let x = 5;
double(x);
console.log(x); // 5 — unchanged

// Object — pass by reference
function rename(obj) { obj.name = "Bob"; }
const user = { name: "Alice" };
rename(user);
console.log(user.name); // "Bob" — mutated!
Q12 What is the difference between for...in and for...of? ▼
  • for...in: Iterates over enumerable property keys of an object (including inherited ones). Use for objects, not arrays.
  • for...of: Iterates over iterable values — arrays, strings, Maps, Sets, generators. Not for plain objects.
const arr = ['a', 'b', 'c'];

for (const key of arr) console.log(key);   // 'a', 'b', 'c' (values)
for (const key in arr) console.log(key);   // '0', '1', '2' (indices)

const obj = { a: 1, b: 2 };
for (const key in obj) console.log(key);   // 'a', 'b'
// for...of on plain object → TypeError

🔵 Scope & Closures — Q13 to Q24

Lexical scope, the scope chain, closures, and the this keyword

Q13 What is scope in JavaScript? What types of scope exist? ▼

Scope defines where variables are accessible. JavaScript uses lexical scoping — a variable's scope is determined by where it is written in the code.

  • Global scope: Accessible everywhere. Avoid polluting it.
  • Function scope: Variables inside a function — only accessible within that function.
  • Block scope: let/const inside {} — accessible only within that block.
  • Module scope: Variables in ES modules are scoped to the module, not global.
const global = "I'm global";

function outer() {
  const outerVar = "outer";
  if (true) {
    let blockVar = "block";   // only inside this {}
    console.log(outerVar);    // accessible (outer scope)
  }
  console.log(blockVar);      // ReferenceError!
}
Q14 What is a Closure in JavaScript? ▼

A closure is a function that remembers the variables from its outer (enclosing) scope even after that scope has finished executing. The inner function "closes over" the outer variables.

function makeCounter() {
  let count = 0;             // outer variable
  return function() {
    count++;                   // inner function closes over count
    return count;
  };
}

const counter = makeCounter();
counter(); // 1
counter(); // 2
counter(); // 3 — count is remembered!

Closures are used for: data privacy (module pattern), factory functions, memoization, event handlers, and currying.

Q15 What is the Scope Chain? ▼

When JavaScript can't find a variable in the current scope, it walks up the chain of parent scopes until it either finds the variable or reaches the global scope (where it throws a ReferenceError).

const x = "global";

function outer() {
  const x = "outer";
  function inner() {
    // no local x — looks up scope chain
    console.log(x); // "outer" (found in parent scope)
  }
  inner();
}
outer();
Q16 What is the Temporal Dead Zone (TDZ)? ▼

The TDZ is the period between entering the scope where a let/const is declared and the point where that declaration is actually initialized. Accessing the variable in this window throws a ReferenceError.

// TDZ starts here for 'name'
console.log(name); // ReferenceError (TDZ!)
let name = "Alice";  // TDZ ends here
console.log(name); // "Alice"
TDZ exists to catch bugs from using variables before they're ready. It's why let/const are safer than var.
Q17 What is 'this' in JavaScript? ▼

this refers to the object that called the function. Its value depends on how the function is called, not where it's defined.

// 1. Global / standalone call
function show() { console.log(this); }
show();          // window (browser) / global (Node) / undefined (strict mode)

// 2. Method call — this = the object
const user = { name: "Alice", greet() { console.log(this.name); } };
user.greet();    // "Alice"

// 3. Arrow function — this = enclosing scope's this
const obj = {
  name: "Bob",
  greet: () => console.log(this.name) // 'this' is NOT obj
};
Q18 What is the difference between call(), apply(), and bind()? ▼

All three explicitly set the value of this for a function:

  • call(thisArg, arg1, arg2): Calls immediately. Args passed individually.
  • apply(thisArg, [args]): Calls immediately. Args passed as array.
  • bind(thisArg, arg1): Returns a NEW function with this permanently bound. Does not call immediately.
function greet(greeting, punct) {
  console.log(`${greeting}, ${this.name}${punct}`);
}
const user = { name: "Alice" };

greet.call(user, "Hello", "!");     // "Hello, Alice!"
greet.apply(user, ["Hi", "."]);    // "Hi, Alice."
const bound = greet.bind(user, "Hey");
bound("?");                        // "Hey, Alice?"
Q19 What is an IIFE (Immediately Invoked Function Expression)? ▼

An IIFE is a function that is defined and called immediately. It creates a private scope to avoid polluting the global namespace — heavily used before ES modules.

// IIFE syntax
(function() {
  const private = "Can't access me outside";
  console.log(private); // works
})();

// Arrow function IIFE
(() => {
  const x = 42;
  console.log(x);
})();

console.log(private); // ReferenceError — private scope!
IIFEs are less common now with ES modules (which have their own scope), but still appear in older codebases and some build tool patterns.
Q20 What is currying in JavaScript? ▼

Currying transforms a function that takes multiple arguments into a sequence of functions each taking one argument. It relies on closures.

// Normal function
const add = (a, b, c) => a + b + c;

// Curried version
const curriedAdd = a => b => c => a + b + c;
curriedAdd(1)(2)(3); // 6

// Partial application — pre-fill args
const addFive = curriedAdd(5);
addFive(3)(2); // 10

// Real use: reusable validators, event handlers
const hasPermission = role => user => user.roles.includes(role);
const isAdmin = hasPermission('admin');
Q21 What is memoization? ▼

Memoization is a caching optimization — a function stores previously computed results so the same computation isn't repeated for the same inputs. Implemented using closures.

function memoize(fn) {
  const cache = new Map();
  return function(...args) {
    const key = JSON.stringify(args);
    if (cache.has(key)) return cache.get(key);
    const result = fn.apply(this, args);
    cache.set(key, result);
    return result;
  };
}

const slowFib = n => n <= 1 ? n : slowFib(n-1) + slowFib(n-2);
const fastFib = memoize(slowFib);
Q22 What is the classic closure-in-loop bug and how do you fix it? ▼ 
// ❌ Bug — all callbacks share the same 'i' reference
for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 100);
}
// Prints: 3, 3, 3

// ✅ Fix 1: use let (block-scoped per iteration)
for (let i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 100);
}
// Prints: 0, 1, 2

// ✅ Fix 2: IIFE to capture each value
for (var i = 0; i < 3; i++) {
  (j => setTimeout(() => console.log(j), 100))(i);
}
Q23 What is the difference between regular functions and arrow functions regarding 'this'? ▼
  • Regular functions: this is dynamic — determined at call time by how the function is called.
  • Arrow functions: this is lexical — inherited from the surrounding code where the arrow function was defined. Arrow functions have no own this.
const timer = {
  seconds: 0,
  startRegular() {
    setInterval(function() {
      this.seconds++; // ❌ 'this' = window, not timer
    }, 1000);
  },
  startArrow() {
    setInterval(() => {
      this.seconds++; // ✅ 'this' = timer (lexical)
    }, 1000);
  }
};
Q24 What is function composition? ▼

Function composition combines two or more functions so that the output of one becomes the input of the next. A core concept in functional programming.

const double    = x => x * 2;
const addOne    = x => x + 1;
const square    = x => x * x;

// Manual composition
square(addOne(double(3)));  // square(addOne(6)) = square(7) = 49

// Generic compose utility (right-to-left)
const compose = (...fns) => x => fns.reduceRight((acc, fn) => fn(acc), x);
const transform = compose(square, addOne, double);
transform(3); // 49

🟡 Asynchronous JavaScript — Q25 to Q36

Event loop, callbacks, promises, async/await, and concurrency patterns

Q25 How does the JavaScript Event Loop work? ▼

JavaScript is single-threaded — it runs one task at a time. The event loop manages async operations via queues:

  • Call Stack: Where synchronous code executes (LIFO)
  • Web APIs / Node APIs: Browser/Node handles async ops (setTimeout, fetch, I/O)
  • Microtask Queue: Resolved promises, queueMicrotask() — processed before macrotasks
  • Macrotask Queue (Task Queue): setTimeout, setInterval, I/O callbacks — processed one per loop iteration
console.log('1');
setTimeout(() => console.log('2'), 0);
Promise.resolve().then(() => console.log('3'));
console.log('4');
// Output: 1, 4, 3, 2
// Sync → Microtask (Promise) → Macrotask (setTimeout)
Q26 What is a callback? What is callback hell? ▼

A callback is a function passed as an argument to another function, to be called when an async operation completes. Callback hell (pyramid of doom) is deeply nested callbacks that become unreadable.

// Callback hell
getUser(id, (user) => {
  getOrders(user.id, (orders) => {
    getOrderDetails(orders[0].id, (details) => {
      sendEmail(details, (result) => {
        console.log("Done"); // 4 levels deep — unmanageable
      });
    });
  });
});

// Fixed with async/await
const user    = await getUser(id);
const orders  = await getOrders(user.id);
const details = await getOrderDetails(orders[0].id);
await sendEmail(details);
Q27 What is a Promise? Explain its states. ▼

A Promise is an object representing the eventual completion or failure of an async operation. It has 3 states:

  • Pending: Initial state — neither fulfilled nor rejected
  • Fulfilled: Operation succeeded — .then() callback is called
  • Rejected: Operation failed — .catch() callback is called
const fetchUser = (id) => new Promise((resolve, reject) => {
  if (id > 0) resolve({ id, name: "Alice" });
  else reject(new Error("Invalid ID"));
});

fetchUser(1)
  .then(user  => console.log(user.name))  // "Alice"
  .catch(err  => console.error(err))
  .finally(() => console.log("Done"));  // always runs
Q28 What is async/await and how does it work? ▼

async/await is syntactic sugar over Promises, making async code look and behave like synchronous code. An async function always returns a Promise. await pauses execution inside the function until the Promise resolves.

async function loadData() {
  try {
    const response = await fetch('/api/data');
    const data = await response.json();
    return data;
  } catch (err) {
    console.error('Failed:', err);
  }
}

// await only works inside async functions
// Or at the top level of ES modules (Top-level await)
const data = await loadData(); // Top-level (ES2022)
Q29 What is the difference between Promise.all, Promise.race, Promise.allSettled, and Promise.any? ▼
  • Promise.all([...]): Resolves when ALL resolve. Rejects immediately if ANY rejects. Results in array of values.
  • Promise.race([...]): Resolves/rejects as soon as the FIRST promise settles (win or lose).
  • Promise.allSettled([...]): Waits for ALL to settle (fulfilled or rejected). Never rejects. Results in array of status objects.
  • Promise.any([...]): Resolves as soon as ANY resolves. Rejects only if ALL reject (AggregateError).
const [users, posts] = await Promise.all([fetchUsers(), fetchPosts()]);

const results = await Promise.allSettled([p1, p2, p3]);
results.forEach(r => console.log(r.status, r.value ?? r.reason));
Q30 What is the difference between microtasks and macrotasks? ▼
  • Microtasks: Processed immediately after the current task, before the next macrotask. Include: resolved Promises (.then/.catch/.finally), queueMicrotask(), MutationObserver.
  • Macrotasks: Processed one per event loop iteration. Include: setTimeout, setInterval, setImmediate (Node), I/O, UI rendering.
setTimeout(() => console.log('macro'), 0);
queueMicrotask(() => console.log('micro'));
Promise.resolve().then(() => console.log('promise'));

// Output: micro, promise, macro
// All microtasks drain before next macrotask
Q31 What is the Fetch API? How does it differ from XMLHttpRequest? ▼

Fetch is a modern Promise-based API for HTTP requests, replacing the older callback-based XMLHttpRequest (XHR).

// Fetch — clean and Promise-based
const data = await fetch('/api/users', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ name: 'Alice' })
}).then(res => {
  if (!res.ok) throw new Error(`HTTP ${res.status}`);
  return res.json();
});
Note: fetch() only rejects on network errors. HTTP 4xx/5xx responses still resolve — you must check res.ok manually.
Q32 What are JavaScript generators? ▼

Generators are functions that can pause (yield) and resume execution. They return an iterator and are useful for lazy sequences, infinite data, and async control flow.

function* range(start, end) {
  for (let i = start; i <= end; i++) {
    yield i;
  }
}

const gen = range(1, 3);
gen.next(); // { value: 1, done: false }
gen.next(); // { value: 2, done: false }
gen.next(); // { value: 3, done: false }
gen.next(); // { value: undefined, done: true }

// Use with for...of
for (const n of range(1, 5)) console.log(n);
Q33 What is an AbortController and when would you use it? ▼

AbortController lets you cancel in-flight fetch requests or async operations — useful for search-as-you-type (cancel previous request) or component cleanup.

const controller = new AbortController();

// Cancel if user types again
inputEl.addEventListener('input', () => {
  controller.abort();
});

try {
  const res = await fetch('/search', { signal: controller.signal });
  const data = await res.json();
} catch (err) {
  if (err.name === 'AbortError') console.log('Cancelled');
}
Q34 What is debouncing and throttling in JavaScript? ▼
  • Debounce: Delays execution until after a pause. Use for search inputs — fire API call only after user stops typing.
  • Throttle: Limits execution to at most once per time window. Use for scroll/resize handlers.
// Debounce — wait for pause
function debounce(fn, delay) {
  let timer;
  return (...args) => {
    clearTimeout(timer);
    timer = setTimeout(() => fn(...args), delay);
  };
}

// Throttle — max once per interval
function throttle(fn, interval) {
  let lastTime = 0;
  return (...args) => {
    const now = Date.now();
    if (now - lastTime >= interval) { lastTime = now; fn(...args); }
  };
}
Q35 What is a Web Worker? ▼

Web Workers run JavaScript in a background thread, separate from the main thread — preventing heavy CPU operations from freezing the UI. Workers communicate via postMessage.

// main.js
const worker = new Worker('worker.js');
worker.postMessage({ data: bigArray });
worker.onmessage = e => console.log('Result:', e.data);

// worker.js (runs in separate thread)
onmessage = (e) => {
  const result = heavyCrunch(e.data.data);
  postMessage(result);
};
Workers cannot access the DOM. Use them for image processing, encryption, sorting large datasets, or any CPU-intensive task.
Q36 What is an Observable vs a Promise? ▼
  • Promise: Single value, one-time. Eager (starts immediately). Not cancellable natively.
  • Observable (RxJS): Multiple values over time (stream). Lazy (starts on subscribe). Cancellable. Powerful operators (map, filter, merge, debounce).
// Promise — one value
const p = fetch('/api').then(r => r.json()); // resolves once

// Observable — stream of values
import { fromEvent } from 'rxjs';
const clicks$ = fromEvent(button, 'click');
const sub = clicks$.subscribe(e => console.log('Clicked!'));
sub.unsubscribe(); // cancel

🟠 OOP & Prototypes — Q37 to Q48

Prototypal inheritance, classes, and object-oriented patterns in JavaScript

Q37 What is the Prototype Chain in JavaScript? ▼

Every JavaScript object has an internal [[Prototype]] link to another object (its prototype). When you access a property, JS first checks the object, then walks up the prototype chain until it finds it or reaches null.

const animal = { eats: true };
const dog = Object.create(animal);
dog.barks = true;

console.log(dog.barks); // true  (own property)
console.log(dog.eats);  // true  (from prototype chain)
console.log(dog.flies); // undefined (not found in chain)

Object.getPrototypeOf(dog) === animal; // true
Q38 What is the difference between __proto__ and prototype? ▼
  • prototype: A property on constructor functions. It's the object that will become the [[Prototype]] of instances created with new.
  • __proto__: A property on instances/objects that points to their prototype. Deprecated — use Object.getPrototypeOf().
function Dog(name) { this.name = name; }
Dog.prototype.bark = function() { console.log("Woof!"); };

const rex = new Dog("Rex");
rex.bark();  // "Woof!" — found via prototype

rex.__proto__ === Dog.prototype  // true
Object.getPrototypeOf(rex) === Dog.prototype // true (preferred)
Q39 How does the 'new' keyword work in JavaScript? ▼

When you call a function with new, JavaScript does 4 things automatically:

  • Creates a new empty object {}
  • Sets its [[Prototype]] to the constructor's .prototype
  • Binds this inside the constructor to the new object
  • Returns the new object (unless the constructor explicitly returns a different object)
function Person(name) {
  // 'this' = new empty object
  this.name = name;
  // return this; (implicit)
}

const alice = new Person("Alice");
alice instanceof Person; // true
Q40 What are ES6 Classes? Are they different from constructor functions? ▼

ES6 class syntax is syntactic sugar over prototype-based inheritance. Under the hood, they still use prototypes — classes are just a cleaner way to write constructor functions.

// Constructor function (old way)
function Animal(name) { this.name = name; }
Animal.prototype.speak = function() { console.log(this.name); };

// ES6 Class (syntactic sugar)
class Animal {
  constructor(name) { this.name = name; }
  speak() { console.log(this.name); }
  static create(name) { return new Animal(name); }
}

class Dog extends Animal {
  speak() { console.log(`${this.name} barks!`); }
  constructor(name) { super(name); } // must call super
}
Q41 What are private class fields? (ES2022) ▼

Private class fields use the # prefix and are only accessible within the class body — true encapsulation in JavaScript.

class BankAccount {
  #balance = 0;        // private field
  #owner;               // private field

  constructor(owner, initial) {
    this.#owner   = owner;
    this.#balance = initial;
  }

  deposit(amount) { this.#balance += amount; }
  get balance()   { return this.#balance; }
}

const acc = new BankAccount("Alice", 100);
acc.deposit(50);
acc.balance;  // 150
acc.#balance; // SyntaxError — private!
Q42 What is Object.create() and how is it different from new? ▼

Object.create(proto) creates a new object with the specified prototype, without calling any constructor. More explicit control over prototype chain.

const proto = {
  greet() { console.log(`Hi, I'm ${this.name}`); }
};

const alice = Object.create(proto);
alice.name = "Alice";
alice.greet(); // "Hi, I'm Alice"

// Object with no prototype (pure dictionary)
const dict = Object.create(null);
// No toString, hasOwnProperty etc — clean slate
Q43 What are getters and setters in JavaScript? ▼

Getters and setters define computed properties — they look like properties but run functions when accessed or assigned. Useful for validation and derived data.

class Temperature {
  #celsius;
  constructor(c) { this.#celsius = c; }

  get fahrenheit() {
    return this.#celsius * 9/5 + 32;
  }
  set fahrenheit(f) {
    this.#celsius = (f - 32) * 5/9;
  }
}

const t = new Temperature(0);
t.fahrenheit;        // 32  (getter)
t.fahrenheit = 212; // setter → celsius becomes 100
Q44 What is a Proxy in JavaScript? ▼

A Proxy wraps an object and intercepts fundamental operations (get, set, delete, etc.) using "traps." Used for validation, logging, reactive systems (Vue 3 uses Proxy for reactivity).

const handler = {
  get(target, prop) {
    console.log(`Getting ${prop}`);
    return prop in target ? target[prop] : `Property ${prop} not found`;
  },
  set(target, prop, value) {
    if (typeof value !== 'number') throw new TypeError('Numbers only!');
    target[prop] = value;
    return true;
  }
};

const proxy = new Proxy({}, handler);
proxy.age = 25;      // OK
proxy.age = "old"; // TypeError
Q45 What is the difference between Object.freeze() and Object.seal()? ▼
  • Object.freeze(obj): Prevents adding, removing, or modifying properties. Makes the object fully immutable (shallow).
  • Object.seal(obj): Prevents adding or removing properties, but existing properties can still be modified.
const frozen = Object.freeze({ x: 1, y: 2 });
frozen.x = 99;   // silently ignored (or TypeError in strict mode)
frozen.z = 3;   // ignored

const sealed = Object.seal({ x: 1, y: 2 });
sealed.x = 99;  // ✅ OK — can modify existing
sealed.z = 3;  // ❌ ignored — can't add new
Both are shallow — nested objects are not frozen/sealed automatically.
Q46 What is a Symbol in JavaScript? ▼

A Symbol is a primitive that creates a guaranteed-unique identifier. Two symbols with the same description are never equal. Used for unique object keys that avoid naming collisions.

const id1 = Symbol('id');
const id2 = Symbol('id');
id1 === id2; // false — always unique

const user = {
  name: 'Alice',
  [id1]: 42  // symbol key — won't clash with 'id' string key
};
user[id1]; // 42

// Well-known symbols
class Range {
  [Symbol.iterator]() { /* make iterable */ }
}
Q47 What is a WeakMap and WeakSet? How are they different from Map and Set? ▼
  • Map/Set: Strong references to keys/values. Prevent garbage collection. Iterable.
  • WeakMap/WeakSet: Keys must be objects. Hold weak references — if the object is garbage-collected, the entry is automatically removed. Not iterable. Useful for caches and private data.
const cache = new WeakMap();

function process(obj) {
  if (cache.has(obj)) return cache.get(obj);
  const result = heavyComputation(obj);
  cache.set(obj, result); // auto-cleaned when obj is GC'd
  return result;
}
Q48 What is the Reflect API? ▼

The Reflect API provides methods for interceptable JavaScript operations. It mirrors the operations that Proxy traps intercept, making it the correct way to perform default behavior inside Proxy handlers.

const handler = {
  set(target, prop, value) {
    console.log(`Setting ${prop} = ${value}`);
    return Reflect.set(target, prop, value); // perform default behavior
  }
};

// Other Reflect methods mirror Object methods
Reflect.ownKeys(obj);       // like Object.getOwnPropertyNames
Reflect.has(obj, 'prop'); // like 'prop' in obj
Reflect.deleteProperty(obj, 'prop'); // like delete obj.prop

🔴 DOM & Events — Q49 to Q57

Document Object Model manipulation and browser event handling

Q49 What is the DOM? How do you select and manipulate elements? ▼

The DOM (Document Object Model) is a tree-like in-memory representation of an HTML document. JavaScript can read and modify it to change the page dynamically.

// Selecting elements
document.getElementById('app');
document.querySelector('.card');         // first match
document.querySelectorAll('li');          // NodeList of all

// Modifying elements
const el = document.querySelector('#title');
el.textContent = 'New Title';
el.innerHTML   = '<strong>Bold</strong>';
el.style.color = 'red';
el.classList.add('active');
el.setAttribute('data-id', '42');

// Creating and appending elements
const li = document.createElement('li');
li.textContent = 'New item';
document.querySelector('ul').appendChild(li);
Q50 What is event bubbling and event capturing? ▼

When an event fires on an element, it travels through the DOM in 3 phases:

  • Capturing phase: Event travels DOWN from document root to the target element.
  • Target phase: Event fires on the target element.
  • Bubbling phase: Event travels UP from the target back to the root.
el.addEventListener('click', handler);       // bubbling (default)
el.addEventListener('click', handler, true);  // capturing

// Stop bubbling
el.addEventListener('click', e => {
  e.stopPropagation(); // don't bubble up
  e.preventDefault();  // prevent default action (e.g. form submit)
});
Q51 What is event delegation? ▼

Instead of adding listeners to every child element, attach ONE listener to the parent and use the event's target to identify which child was clicked. More efficient — especially for dynamic lists.

// ❌ Inefficient — listener on every item
document.querySelectorAll('li').forEach(li =>
  li.addEventListener('click', handler)
);

// ✅ Delegation — one listener on parent
document.querySelector('ul').addEventListener('click', e => {
  if (e.target.tagName === 'LI') {
    console.log('Clicked:', e.target.textContent);
  }
});
Q52 What is the difference between innerHTML, textContent, and innerText? ▼
  • innerHTML: Gets/sets raw HTML markup including tags. Risk of XSS if used with user input.
  • textContent: Gets/sets plain text only — strips all tags. Safe, fast, includes hidden elements.
  • innerText: Gets/sets visible text. Triggers layout reflow (slow). Respects CSS visibility and line breaks.
el.innerHTML   = '<b>Bold</b>'; // rendered as bold
el.textContent = '<b>Bold</b>'; // shows literal "<b>Bold</b>"
el.innerText   = 'Hello';       // visible text only
Never use el.innerHTML = userInput without sanitization — it can execute malicious scripts (XSS).
Q53 What is the difference between window.onload and DOMContentLoaded? ▼
  • DOMContentLoaded: Fires when the HTML is fully parsed and the DOM is ready — before images, stylesheets, and iframes finish loading. Faster.
  • window.load (or load event): Fires when the ENTIRE page is loaded including all resources (images, fonts, CSS). Slower.
document.addEventListener('DOMContentLoaded', () => {
  // DOM ready — safe to query elements
  initApp();
});

window.addEventListener('load', () => {
  // Everything loaded — images, fonts etc.
  hideLoadingScreen();
});
Q54 What is the difference between localStorage, sessionStorage, and cookies? ▼
  • localStorage: Persists until explicitly cleared. ~5-10MB. Not sent to server. String values only.
  • sessionStorage: Cleared when browser tab is closed. Same size limit. Not sent to server.
  • Cookies: Sent to server with every request. Smaller (~4KB). Can set expiry. Support HttpOnly (no JS access) and Secure flags.
localStorage.setItem('theme', 'dark');
localStorage.getItem('theme');     // 'dark'
localStorage.removeItem('theme');

// Store objects
localStorage.setItem('user', JSON.stringify({ name: 'Alice' }));
const user = JSON.parse(localStorage.getItem('user'));
Q55 What is requestAnimationFrame? ▼

requestAnimationFrame(callback) schedules a function to run before the next browser repaint — synchronized with the display refresh rate (typically 60fps). The correct way to write smooth animations.

function animate(timestamp) {
  // Update animation state
  element.style.left = Math.sin(timestamp / 1000) * 100 + 'px';
  requestAnimationFrame(animate); // schedule next frame
}

requestAnimationFrame(animate); // start
Use requestAnimationFrame instead of setInterval for animations — it pauses when the tab is hidden and is much more efficient.
Q56 What is the Intersection Observer API? ▼

The Intersection Observer API watches when an element enters or exits the viewport — without expensive scroll event listeners. Used for lazy loading, infinite scroll, and animations on scroll.

const observer = new IntersectionObserver(entries => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      entry.target.classList.add('visible');
      observer.unobserve(entry.target); // stop watching
    }
  });
}, { threshold: 0.1 }); // 10% visible

document.querySelectorAll('.card').forEach(el => observer.observe(el));
Q57 What is a MutationObserver? ▼

A MutationObserver watches for changes to the DOM tree (added/removed nodes, attribute changes, text changes) and fires a callback when they occur.

const observer = new MutationObserver(mutations => {
  mutations.forEach(mutation => {
    console.log('Changed:', mutation.type, mutation.addedNodes);
  });
});

observer.observe(document.body, {
  childList: true,  // watch for child nodes added/removed
  subtree: true,     // watch all descendants
  attributes: true   // watch attribute changes
});

🟣 ES6+ Features — Q58 to Q68

Modern JavaScript syntax and features from ES2015 onwards

Q58 What is destructuring in JavaScript? ▼

Destructuring lets you unpack values from arrays or properties from objects into distinct variables with concise syntax.

// Array destructuring
const [a, b, ...rest] = [1, 2, 3, 4];
// a=1, b=2, rest=[3,4]

const [, second] = [1, 2]; // skip first

// Object destructuring
const { name, age = 25, address: { city } } = user; // default + nested

// Rename while destructuring
const { name: userName } = user;

// In function parameters
function display({ name, age }) { console.log(name, age); }
Q59 What are template literals? ▼

Template literals use backticks and support multi-line strings and embedded expressions with ${} syntax.

const name = "Alice";
const age = 30;

// Interpolation
const msg = `Hello, ${name}! You are ${age > 18 ? 'adult' : 'minor'}.`;

// Multi-line
const html = `
  <div>
    <h1>${name}</h1>
  </div>
`;

// Tagged templates (advanced)
function safe(strings, ...values) { /* sanitize values */ }
const query = safe`SELECT * FROM users WHERE id = ${userId}`;
Q60 What is the spread operator and rest parameters? ▼
  • Spread (...): Expands an iterable into individual elements. Used in function calls, array/object literals.
  • Rest (...): Collects remaining arguments into an array. Used in function parameters.
// Spread — expand
const arr = [1, 2, 3];
Math.max(...arr);        // 3
const copy = [...arr, 4]; // [1,2,3,4]
const merged = { ...obj1, ...obj2 }; // shallow merge

// Rest — collect
function sum(first, ...rest) {
  return first + rest.reduce((a, b) => a + b, 0);
}
sum(1, 2, 3, 4); // 10
Q61 What are ES6 Modules? How are they different from CommonJS? ▼
  • ES Modules (ESM): import/export. Static analysis (tree-shakeable). Async loading. Runs in strict mode. Native in browsers and Node.js.
  • CommonJS (CJS): require()/module.exports. Synchronous. Dynamic — can require inside conditionals. Node.js default (pre-ESM).
// ES Modules
export const PI = 3.14;
export default function add(a, b) { return a + b; }

import add, { PI } from './math.js';
import * as Math from './math.js';

// Dynamic import (lazy loading)
const module = await import('./heavy.js');
module.default();
Q62 What are Map and Set in JavaScript? ▼
  • Map: Like an object but keys can be ANY type (functions, objects, primitives). Maintains insertion order. Has a size property.
  • Set: A collection of UNIQUE values (no duplicates). Maintains insertion order. O(1) lookup.
// Map
const map = new Map();
map.set('key', 'value');
map.set(obj, 42);         // object as key!
map.get('key');          // 'value'
map.size;                  // 2

// Set — deduplicate array
const unique = [...new Set([1, 2, 2, 3, 3])]; // [1,2,3]
const set = new Set(['a', 'b', 'a']);
set.has('a'); // true
set.size;     // 2
Q63 What is optional chaining (?.) and nullish coalescing (??)? ▼

Optional chaining (?.): Safely access deeply nested properties — returns undefined instead of throwing a TypeError if any part is null/undefined.

Nullish coalescing (??): Returns right side only if left side is null or undefined (unlike || which also triggers for 0, "", false).

// Optional chaining
user?.address?.city          // undefined if any is null
arr?.[0]                     // safe array access
obj?.method?.()              // safe method call

// Nullish coalescing
const count = data.count ?? 0; // 0 only if null/undefined
const name  = user?.name ?? 'Guest';

// ??= (nullish assignment, ES2021)
user.settings ??= {}; // assign only if null/undefined
Q64 What are iterators and iterables? ▼

An iterable is any object with a [Symbol.iterator]() method (arrays, strings, Maps, Sets). An iterator is an object with a next() method returning { value, done }.

// Custom iterable range
const range = {
  from: 1, to: 5,
  [Symbol.iterator]() {
    let current = this.from;
    const last = this.to;
    return {
      next() {
        return current <= last
          ? { value: current++, done: false }
          : { value: undefined, done: true };
      }
    };
  }
};

for (const n of range) console.log(n); // 1,2,3,4,5
[...range]; // [1,2,3,4,5]
Q65 What is the difference between Array.from() and the spread operator? ▼
  • Both convert iterables to arrays, but Array.from() also accepts a mapping function and works on array-like objects (e.g., NodeList, arguments) that aren't iterable.
// Both work on iterables
[...new Set([1,2,2])];         // [1,2]
Array.from(new Set([1,2,2]));  // [1,2]

// Array.from with map fn
Array.from({ length: 5 }, (_, i) => i * 2); // [0,2,4,6,8]

// Array-like (not iterable) — only Array.from works
Array.from(document.querySelectorAll('li'));
// [...nodeList] also works since NodeList IS iterable
Q66 What are common array methods in modern JavaScript? ▼ 
[1,2,3].map(x => x * 2)          // [2,4,6]
[1,2,3,4].filter(x => x > 2)    // [3,4]
[1,2,3].reduce((a, b) => a+b, 0) // 6
[1,2,3].find(x => x > 1)         // 2 (first match)

Q67 What are tagged template literals? ▼

A tagged template is a function placed before a template literal. The function receives the string parts and the interpolated values separately, enabling powerful custom processing.

function highlight(strings, ...values) {
  return strings.reduce((result, str, i) => {
    const val = values[i - 1];
    return result + (typeof val === 'number'
      ? `<b>${val}</b>`
      : val ?? '') + str;
  });
}

const age = 25;
highlight`User is ${age} years old`;
// "User is <b>25</b> years old"

⚫ Advanced JavaScript — Q69 to Q77

Memory, performance, design patterns, and engine internals

Q69 How does garbage collection work in JavaScript? ▼

JavaScript uses automatic memory management via the Mark-and-Sweep algorithm. The garbage collector periodically marks all reachable objects (starting from roots like global variables, the call stack), then sweeps away unmarked (unreachable) objects.

  • An object is reachable if it can be accessed via a chain of references from a root
  • Once no references point to an object, it becomes eligible for collection
Modern V8 uses generational GC (Young/Old generation), incremental marking, and concurrent sweeping to minimize pauses.
Q70 What is a memory leak in JavaScript? How do you find and fix them? ▼

A memory leak occurs when memory that is no longer needed is not released — usually because unintended references keep objects alive.

Common causes:

  • Global variables — never garbage collected
  • Forgotten event listeners — not removed when component unmounts
  • Closures holding large data unnecessarily
  • DOM references to removed elements
  • setInterval that never gets cleared
// ❌ Leak — listener never removed
window.addEventListener('resize', heavyHandler);

// ✅ Fix — remove when done
window.addEventListener('resize', heavyHandler);
// On cleanup:
window.removeEventListener('resize', heavyHandler);
Find leaks: Chrome DevTools → Memory → Take heap snapshots and compare. Look for objects growing over time.
Q71 What is the V8 engine? What is JIT compilation? ▼

V8 is Google's open-source JavaScript engine (used in Chrome and Node.js). It compiles JavaScript to native machine code using JIT (Just-In-Time) compilation instead of interpreting it line by line.

  • Parser: Parses JS source into an AST (Abstract Syntax Tree)
  • Ignition (Interpreter): Compiles AST to bytecode and executes it
  • TurboFan (Optimizing Compiler): Identifies "hot" code paths and compiles them to highly optimized machine code
  • Deoptimization: If assumptions change (e.g., type changes), V8 deoptimizes back to bytecode
Write consistent types in hot code paths to help V8 optimize effectively (avoid hidden class changes).
Q72 What is tail call optimization? ▼

A tail call is when a function's last action is calling another function. In tail call optimization (TCO), the engine reuses the current stack frame instead of creating a new one, preventing stack overflows in deep recursion.

// NOT tail call — must keep frame for * operation
function factorial(n) {
  if (n === 1) return 1;
  return n * factorial(n - 1); // not last operation
}

// Tail call — accumulator pattern
function factorial(n, acc = 1) {
  if (n === 1) return acc;
  return factorial(n - 1, n * acc); // last operation is the call
}
TCO is only specified in strict mode and is inconsistently implemented across engines. Safari implements it; V8 does not yet.
Q73 What is the Module Pattern in JavaScript? ▼

The Module Pattern uses closures (often IIFEs) to create private state and expose a public API. It was the standard way to encapsulate code before ES modules.

const CartModule = (() => {
  // Private
  let items = [];

  // Public API
  return {
    add(item)    { items.push(item); },
    remove(id)   { items = items.filter(i => i.id !== id); },
    getCount()   { return items.length; },
    getTotal()   { return items.reduce((s, i) => s + i.price, 0); }
  };
})();

CartModule.add({ id: 1, price: 9.99 });
CartModule.getCount(); // 1
Q74 What is the Observer Pattern in JavaScript? ▼

The Observer Pattern defines a one-to-many dependency — when one object changes state, all its subscribers are notified. The basis for event emitters and reactive programming.

class EventEmitter {
  #listeners = new Map();

  on(event, fn) {
    if (!this.#listeners.has(event)) this.#listeners.set(event, []);
    this.#listeners.get(event).push(fn);
  }
  emit(event, ...args) {
    this.#listeners.get(event)?.forEach(fn => fn(...args));
  }
  off(event, fn) {
    this.#listeners.set(event, this.#listeners.get(event)?.filter(f => f !== fn));
  }
}

const bus = new EventEmitter();
bus.on('login', user => console.log(`Welcome ${user.name}`));
bus.emit('login', { name: 'Alice' }); // "Welcome Alice"
Q75 What are JavaScript design patterns? Name the most important ones. ▼
  • Singleton: One instance only — global state management, DB connection
  • Factory: Creates objects without specifying exact class — hides creation complexity
  • Observer/PubSub: Event-driven communication between components
  • Module: Encapsulate private state, expose public API
  • Decorator: Add behavior to objects dynamically
  • Strategy: Swap algorithms at runtime (sorting, validation, payment methods)
  • Command: Encapsulate operations as objects — enables undo/redo
  • Proxy: Intercept and control access to objects
Q76 What is tree shaking and dead code elimination? ▼

Tree shaking is a build-time optimization where bundlers (Webpack, Rollup, Vite) remove unused code from the final bundle. It works because ES modules have static imports — the bundler can determine exactly what's used at build time.

// math.js — exports 3 functions
export const add    = (a, b) => a + b;
export const sub    = (a, b) => a - b;
export const square = n => n * n;

// app.js — only imports add
import { add } from './math.js';

// After tree shaking: sub and square are removed from bundle
Avoid barrel files (index.js re-exporting everything) — they often defeat tree shaking. Use direct imports instead.
