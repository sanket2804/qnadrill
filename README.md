# QnADrill

QnADrill is a static interview-preparation website made of topic-based HTML guides. Each guide contains categorized questions and expandable answers for quick revision.

## Included Guides

- `index.html` - landing page for all available guides
- `php.html` - Core PHP interview guide
- `javascript.html` - JavaScript interview guide
- `reactjs.html` - React.js interview guide
- `nodejs.html` - Node.js interview guide
- `common.js` - shared top navigation and common menu-bar styling

## Features

- Static HTML pages with no build step
- Shared navigation menu across all pages
- Topic-based tab sections inside each guide
- Expandable question-and-answer blocks
- Mobile-friendly layout

## How To Run

Because this is a plain static site, you can open `index.html` directly in a browser.

If you want to serve it locally with a simple server, you can also run one of these from the `qnadrill` folder:

```powershell
python -m http.server 8000
```

Then open:

```text
http://localhost:8000
```

## Shared Navigation

The top menu is generated by `common.js`.

Each page sets a page key before loading the shared script:

```html
<script>
  window.QNADRILL_PAGE = 'php';
</script>
<script src="common.js" defer></script>
```

Current supported page keys:

- `home`
- `php`
- `javascript`
- `react`
- `node`

`common.js` uses that key to:

- inject the shared menu bar
- highlight the active page
- apply the correct accent color for the current guide

## Adding A New Guide

1. Create a new HTML file in the `qnadrill` folder.
2. Add the shared page identifier before `common.js`.
3. Add a link for the new page inside `common.js`.
4. Add a card for the new guide in `index.html`.
5. Update the landing-page stats if needed.

Example setup for a new guide:

```html
<script>
  window.QNADRILL_PAGE = 'newtopic';
</script>
<script src="common.js" defer></script>
```

## Design Notes

- Each guide keeps its own page-specific styles for header color, tabs, cards, and content layout.
- Only the top navigation is shared globally through `common.js`.
- The landing page uses the same visual language as the topic guides for a consistent experience.

## Maintenance Tips

- Edit `common.js` when you want to change the shared menu across the whole project.
- Edit each HTML file only for guide-specific content or styling.
- Keep guide filenames simple and consistent because navigation links are hardcoded.
