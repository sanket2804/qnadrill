🟢 Fresher Questions — Q1 to Q20

Core Node.js fundamentals expected from entry-level candidates

Q1 How does Node.js function? What is it? ▼

Node.js is an open-source, cross-platform runtime environment that allows JavaScript to run on the server side using Google's V8 engine. Unlike browser JS, Node.js can access the file system, network, and OS resources.

It works using a single-threaded event loop with non-blocking I/O, meaning it can handle thousands of concurrent connections without creating a new thread for each.

const http = require('http');
const server = http.createServer((req, res) => {
  res.end('Hello from Node.js!');
});
server.listen(3000);
Interview tip: Mention V8 engine, event-driven, non-blocking I/O, and that it uses JavaScript outside the browser.
Q2 What characteristics does Node.js have? ▼
  • Asynchronous & Non-blocking: Doesn't wait for I/O operations to complete
  • Single-threaded: Uses one thread with the event loop
  • Fast execution: Powered by Chrome's V8 engine
  • Scalable: Handles many connections simultaneously
  • NPM ecosystem: Access to 1M+ open-source packages
  • Cross-platform: Runs on Windows, Linux, macOS
Q3 Describe NPM. ▼

NPM (Node Package Manager) is the default package manager for Node.js. It lets you install, share, and manage third-party JavaScript libraries (packages) in your project.

# Install a package
npm install express

# Install globally
npm install -g nodemon

# Initialize a project
npm init

All project dependencies are listed in package.json. NPM hosts over 1.8 million packages at npmjs.com.

Q4 What does a Node.js event loop do? ▼

The event loop is the core mechanism that allows Node.js to perform non-blocking I/O despite being single-threaded. It continuously checks the call stack and the callback queue.

  • If the call stack is empty, it picks the next callback from the queue and pushes it to the stack
  • I/O tasks (file reads, HTTP requests) are offloaded to the OS/libuv thread pool
  • When complete, their callbacks are queued and picked up by the event loop
// This demonstrates async non-blocking behavior
console.log('Start');
setTimeout(() => console.log('Timer'), 0);
console.log('End');
// Output: Start → End → Timer
Q5 In Node.js, what is a callback? ▼

A callback is a function passed as an argument to another function, which is then called when an asynchronous operation completes.

const fs = require('fs');
fs.readFile('file.txt', 'utf8', (err, data) => {
  if (err) throw err;
  console.log(data); // Called when file is read
});
Callback Hell: Deeply nested callbacks become hard to read — this problem is solved by Promises and async/await.
Q6 In Node.js, what is a Promise? ▼

A Promise is an object representing the eventual completion or failure of an async operation. It has 3 states: pending, fulfilled, or rejected.

const fetchData = new Promise((resolve, reject) => {
  setTimeout(() => resolve('Data!'), 1000);
});

fetchData
  .then(data => console.log(data))  // 'Data!'
  .catch(err => console.error(err));

Promises are cleaner than callbacks and can be chained. async/await is syntactic sugar built on top of Promises.

Q7 What kinds of streams does Node.js support? ▼

Node.js supports 4 types of streams:

  • Readable: Used to read data (e.g., fs.createReadStream())
  • Writable: Used to write data (e.g., fs.createWriteStream())
  • Duplex: Both readable and writable (e.g., TCP socket)
  • Transform: Duplex stream that modifies data as it passes through (e.g., zlib compression)
const fs = require('fs');
const read = fs.createReadStream('input.txt');
const write = fs.createWriteStream('output.txt');
read.pipe(write); // Efficient piping
Q8 What distinguishes blocking code from non-blocking code? ▼

Blocking code halts execution until the operation completes. Non-blocking code continues execution and handles the result via a callback/promise when ready.

// Blocking (synchronous)
const data = fs.readFileSync('file.txt'); // Waits here
console.log(data);

// Non-blocking (asynchronous)
fs.readFile('file.txt', (err, data) => {
  console.log(data); // Called later
});
console.log('This runs immediately');
Node.js is designed for non-blocking code. Use sync methods only during startup/initialization, not in request handlers.
Q9 What does a Node.js module do? ▼

A module is a reusable block of code that encapsulates related functionality. Node.js uses the CommonJS module system where each file is its own module.

// math.js — define and export
const add = (a, b) => a + b;
module.exports = { add };

// app.js — import and use
const { add } = require('./math');
console.log(add(2, 3)); // 5

Node.js has three types: Core modules (fs, http, path), Third-party modules (via npm), and Local modules (your own files).

Q11 How does Node.js read and write files? ▼

Node.js uses the built-in fs (File System) module for file operations.

Q12 How does Express.js function, and what is it? ▼

Express.js is a minimal, unopinionated web framework for Node.js. It simplifies creating web servers and APIs by providing routing, middleware support, and request/response helpers.

const express = require('express');
const app = express();

app.get('/', (req, res) => {
  res.send('Hello World!');
});

app.listen(3000, () => console.log('Server running'));
Q13 What does Express.js middleware do? ▼

Middleware are functions that have access to the request (req), response (res), and the next function. They execute in sequence and can modify req/res or end the cycle.

// Logger middleware
app.use((req, res, next) => {
  console.log(`${req.method} ${req.url}`);
  next(); // Pass to next middleware
});

// Built-in middleware for JSON parsing
app.use(express.json());

Common uses: authentication, logging, parsing request bodies, error handling, CORS.

Q14 What distinguishes PUT from POST in Express.js? ▼
  • POST: Creates a new resource. Not idempotent — calling it twice creates two records.
  • PUT: Updates/replaces an existing resource. Idempotent — calling it twice gives the same result.
app.post('/users', createUser);       // Create
app.put('/users/:id', updateUser);    // Full update
app.patch('/users/:id', patchUser);  // Partial update
Q15 RESTful API: What is it? ▼

REST (Representational State Transfer) is an architectural style for designing networked APIs. A RESTful API uses standard HTTP methods and is stateless.

  • GET /users → Fetch all users
  • POST /users → Create user
  • PUT /users/1 → Update user 1
  • DELETE /users/1 → Delete user 1

Key principles: Stateless, Client-Server separation, Uniform Interface, Cacheable responses.

Q16 In Node.js, what is Socket.io? ▼

Socket.io is a library that enables real-time, bidirectional communication between the server and browser clients using WebSockets (with fallbacks like long-polling).

// Server
const io = require('socket.io')(server);
io.on('connection', socket => {
  socket.emit('message', 'Welcome!');
  socket.on('chat', msg => io.emit('chat', msg));
});

Used for: live chat, real-time notifications, multiplayer games, collaborative tools.

Q17 How does Node.js handle errors? ▼
  • try/catch: For synchronous code and async/await
  • Error-first callbacks: (err, data) => {}
  • .catch(): For Promises
  • EventEmitter error events: emitter.on('error', handler)
  • process.on('uncaughtException'): Global fallback (not recommended as primary)
async function getData() {
  try {
    const data = await fetchFromDB();
    return data;
  } catch (err) {
    console.error('DB error:', err.message);
  }
}

Q19 What does Node.js clustering mean? ▼

The cluster module allows you to create child processes (workers) that share the same server port. This takes advantage of multi-core CPUs since Node.js is single-threaded by default.

const cluster = require('cluster');
const os = require('os');

if (cluster.isMaster) {
  os.cpus().forEach(() => cluster.fork()); // One worker per CPU
} else {
  require('./app'); // Each worker runs the app
}

  • Production: Minified errors, performance optimizations, caching enabled, security hardened

# Check in code

  // Production-only settings
}

🔵 Comparative Questions — Q21 to Q30

Head-to-head technology comparisons

Q21 What is the difference between Node.js and JavaScript? ▼
  • JavaScript: A programming language that runs in browsers. Handles DOM manipulation, events, and UI logic.
  • Node.js: A runtime environment that lets JavaScript run on the server. Provides access to the file system, network, OS, and databases.
Think of JavaScript as the language, Node.js as the engine that runs it outside the browser.
Q22 How does Node.js differ from traditional server-side scripting languages like PHP? ▼
  • Node.js: Event-driven, non-blocking I/O — handles many requests concurrently on a single thread
  • PHP: Blocking — each request spawns a new thread/process, consuming more memory
  • Node.js excels at real-time apps and APIs; PHP is battle-tested for traditional web pages (WordPress, Laravel)
  • Node.js uses JavaScript everywhere (frontend + backend); PHP is backend-only
Q23 How does Node.js compare to other server-side JavaScript frameworks like Express? ▼

Node.js is the runtime — the foundation. Express.js is a framework built on top of Node.js that adds structure for routing, middleware, and request handling.

Node.js alone can create HTTP servers, but Express makes it much simpler and more organized. Other frameworks built on Node.js include Koa, Hapi, Fastify, and NestJS.

Q24 What is the difference between Node.js and Deno? ▼
  • Security: Deno is secure by default — scripts can't access file system or network without explicit permission. Node.js has no such sandbox.
  • TypeScript: Deno supports TypeScript natively. Node.js needs extra setup (ts-node or compilation).
  • Modules: Deno uses ES modules and URL imports; Node.js uses CommonJS (require) with npm.
  • Package manager: Deno has no npm — imports are URL-based. Node.js uses npm/yarn.
  • Both use the V8 engine and were created by the same person (Ryan Dahl).
Q25 How does Node.js differ from traditional multi-threaded server-side programming languages? ▼
  • Java/C#: Create a new thread per request → high memory usage for concurrent connections
  • Node.js: Single thread + event loop → handles thousands of connections with minimal memory
  • Node.js is ideal for I/O-heavy tasks (APIs, chat). Multi-threaded languages are better for CPU-heavy tasks (image processing, ML).
Q26 What is the difference between Node.js and AJAX? ▼
  • AJAX: A browser-side technique for making async HTTP requests without refreshing the page. Runs in the browser using JavaScript's XMLHttpRequest or fetch().
  • Node.js: A server-side runtime environment. It processes the request that AJAX sends.

AJAX sends a request → Node.js server receives and processes it → sends back a response → AJAX updates the UI.

Q27 How does Node.js compare to PHP in terms of performance? ▼

Node.js is generally faster than PHP for handling high-concurrency I/O workloads because of its non-blocking event loop. PHP creates a new process per request which is slower under heavy load.

However, PHP 8+ with JIT compilation has closed the gap significantly. For traditional server-rendered HTML pages, PHP can be equally performant.

Node.js shines with real-time apps, streaming, and APIs. PHP still dominates CMS-driven websites.
Q28 What is the difference between Node.js and Ruby on Rails? ▼
  • Language: Node.js uses JavaScript; Rails uses Ruby
  • Architecture: Node.js is a runtime (you choose your framework); Rails is an opinionated full-stack MVC framework
  • Speed: Node.js is faster for I/O-heavy tasks; Rails has faster development speed due to conventions
  • Use case: Node.js for real-time APIs; Rails for rapid CRUD app development
Q29 How does Node.js compare to Python in terms of popularity? ▼

Both are extremely popular. Node.js surged in popularity for web APIs and real-time apps. Python dominates in data science, ML, and scripting.

  • Node.js: Preferred for microservices, real-time apps, full-stack JS development
  • Python: Preferred for AI/ML, data analysis, automation, and scripting

In web development rankings (Stack Overflow survey), Node.js consistently ranks as the most-used technology.

Q30 What is the difference between Node.js and Angular? ▼
  • Node.js: Server-side runtime. Handles backend logic, databases, and APIs.
  • Angular: Client-side (frontend) framework for building single-page applications in the browser using TypeScript.

They are often used together: Angular handles the UI, Node.js (with Express) handles the backend API. This combination is part of the MEAN stack (MongoDB + Express + Angular + Node).

🟡 Intermediate Questions — Q31 to Q50

Practical concepts expected from mid-level Node.js developers

Q31 What is Node.js, and how is it different from other server-side technologies? ▼

Node.js is a JavaScript runtime built on Chrome's V8 engine for server-side development. Unlike PHP or Java which use blocking, thread-per-request models, Node.js uses a single-threaded, non-blocking, event-driven architecture that handles concurrency efficiently through the event loop.

Q32 Explain the concept of event-driven programming in Node.js. ▼

Event-driven programming means the flow of execution is determined by events (user actions, messages, I/O completion). Node.js uses an EventEmitter class to emit and listen to events.

const EventEmitter = require('events');
const emitter = new EventEmitter();

emitter.on('greet', (name) => {
  console.log(`Hello, ${name}!`);
});

emitter.emit('greet', 'Alice'); // Hello, Alice!
Q33 How do you handle errors in Node.js? ▼

Best practices for error handling:

// 1. async/await with try/catch
async function run() {
  try { await riskyOp(); }
  catch (err) { console.error(err); }
}

// 2. Express error middleware
app.use((err, req, res, next) => {
  res.status(500).json({ error: err.message });
});

Always handle promise rejections and never let uncaught exceptions crash a production server without logging.

Q34 How do you create a server in Node.js? ▼ 
// Using core http module
const http = require('http');
const server = http.createServer((req, res) => {
  res.writeHead(200, { 'Content-Type': 'text/plain' });

server.listen(3000, () => console.log('Listening on port 3000'));

// Or with Express (simpler)

Q36 What are streams in Node.js? ▼

Streams process data piece by piece (chunks) instead of loading it all into memory. Ideal for large files or network data.

const fs = require('fs');
const zlib = require('zlib');

// Stream + Transform: compress a file
fs.createReadStream('input.txt')
  .pipe(zlib.createGzip())
  .pipe(fs.createWriteStream('input.txt.gz'));

Types: Readable, Writable, Duplex, Transform. The .pipe() method chains streams together.

Q37 What is middleware in Node.js? ▼

Middleware functions run between receiving the request and sending the response. They form a pipeline and must call next() to pass control forward.

Q40 Explain the concept of callbacks in Node.js. ▼

A callback is a function passed as an argument that gets executed after an async operation completes. Node.js follows the error-first callback convention.

fs.readFile('file.txt', (err, data) => {
  if (err) {
    console.error('Error:', err);
    return;
  }
  console.log(data.toString());
});

The first argument is always the error (or null), the second is the result. Deeply nested callbacks create "callback hell" — solved by Promises and async/await.

Q41 How do you implement authentication and authorization in a Node.js application? ▼

Authentication = verifying who you are. Authorization = verifying what you can do.

// JWT Authentication
const jwt = require('jsonwebtoken');

// Login: create token
const token = jwt.sign({ userId: user.id }, process.env.JWT_SECRET, { expiresIn: '1h' });

// Middleware: verify token
const authMiddleware = (req, res, next) => {
  const token = req.headers.authorization?.split(' ')[1];
  const decoded = jwt.verify(token, process.env.JWT_SECRET);
  req.user = decoded;
  next();
};

Popular libraries: Passport.js, jsonwebtoken, bcrypt for password hashing.

Q43 How do you handle database connections in Node.js? ▼

Always store credentials in environment variables, never hardcode them.

Q44 What is the purpose of the Express.js framework in Node.js? ▼

Express.js simplifies building web servers and REST APIs with Node.js by providing: routing, middleware support, template rendering, and request/response helpers. Without Express, you'd manually parse URLs, headers, and bodies using the raw http module.

Q45 Explain the concept of asynchronous programming in Node.js. ▼

Asynchronous programming means operations are initiated and the program continues running — the result is handled when it arrives (callback, promise, or await).

// Evolution of async in Node.js

// 1. Callback style
fs.readFile('f.txt', cb);

// 2. Promise style
fs.promises.readFile('f.txt').then(cb);

// 3. async/await (modern, recommended)
const data = await fs.promises.readFile('f.txt');

Q47 What is the difference between process.nextTick() and setImmediate() in Node.js? ▼
  • process.nextTick(): Executes callback at the end of the current operation, before the event loop continues — highest priority
  • setImmediate(): Executes callback in the next iteration of the event loop (check phase)
process.nextTick(() => console.log('nextTick'));
setImmediate(() => console.log('setImmediate'));
console.log('sync');
// Output: sync → nextTick → setImmediate
Use process.nextTick() for operations that must happen before I/O events. Use setImmediate() for deferred work after I/O.

🔴 Advanced Questions — Q51 to Q60

Deep-dive architecture and design questions for senior developers

Q51 What are streams in Node.js, and how can they be used? ▼

Streams process data in chunks, making them memory-efficient for large datasets. Instead of loading a 1GB file entirely into RAM, streams read/write it piece by piece.

// Efficient file copy using streams
const { pipeline } = require('stream/promises');
await pipeline(
  fs.createReadStream('large.csv'),
  transformStream,
  fs.createWriteStream('output.csv')
);

Real use cases: video streaming, file uploads, HTTP response streaming, log processing.

Q52 What is clustering in Node.js, and how can it be used to improve performance? ▼

Node.js is single-threaded, so a 16-core server uses only 1 core by default. Clustering creates multiple worker processes that share the server port, fully utilizing hardware.

const cluster = require('cluster');
const numCPUs = require('os').cpus().length;

if (cluster.isPrimary) {
  for (let i = 0; i < numCPUs; i++) cluster.fork();
  cluster.on('exit', () => cluster.fork()); // Auto-restart
} else {
  require('./server'); // Each worker runs independently
}
PM2 handles clustering automatically: pm2 start app.js -i max
Q53 What are the differences between require and import statements in Node.js? ▼
  • require (CommonJS): Synchronous, loads at runtime, can be conditional, works in all Node versions
  • import (ES Modules): Asynchronous, static (analyzed at parse time), tree-shakeable, requires "type": "module" in package.json or .mjs extension
// CommonJS
const express = require('express');

// ES Modules
import express from 'express';
ES Modules are the modern standard. Use them for new projects if your Node version supports it (v12+).
Q54 How does Node.js handle asynchronous code execution, and what are the best practices? ▼

Node.js delegates I/O operations to libuv (a C library with a thread pool). When complete, callbacks are queued and the event loop picks them up.

Best practices:

  • Use async/await over nested callbacks
  • Use Promise.all() for concurrent independent operations
  • Never block the event loop with heavy CPU work — use Worker Threads
  • Always handle errors in async functions
// Run in parallel — faster than sequential await
const [users, posts] = await Promise.all([
  fetchUsers(),
  fetchPosts()
]);
Q55 What is the role of the module object in Node.js, and how can it be used to create reusable code? ▼

Every file in Node.js is wrapped in a Module object. It has properties like module.exports, module.id, module.filename, and module.parent.

// module.exports — expose what you want public
class UserService {
  getAll() { return db.findAll('users'); }
}
module.exports = new UserService(); // Singleton pattern

// Because Node caches modules, this is always the same instance
Q56 What are the different types of Node.js modules, and how can they be used to build scalable applications? ▼
  • Core modules: Built-in (fs, http, path, os, crypto, events) — no install needed
  • Third-party modules: Installed via npm (express, mongoose, jest)
  • Local modules: Your own files (./utils, ./models)

For scalable apps: organize code into feature-based modules (routes, controllers, services, models), each with a single responsibility. Use dependency injection to keep modules testable.

Q57 How does Node.js handle errors, and what are some common error-handling techniques? ▼ 
// 1. Custom Error class
class AppError extends Error {

    this.statusCode = statusCode;
  }

// 2. Global Express error handler

Q58 How can Node.js be used to create real-time applications, such as chat apps or dashboards? ▼

Use WebSockets via Socket.io or the native ws library for persistent bidirectional connections.

// Chat app with Socket.io
io.on('connection', (socket) => {
  socket.on('message', (msg) => {
    io.emit('message', { user: socket.id, text: msg });
  });
  socket.on('disconnect', () => {
    console.log('User left');
  });
});

For dashboards: use Server-Sent Events (SSE) for one-way real-time updates without full WebSocket overhead.

Q59 What is GraphQL, and how can it be used with Node.js to build APIs? ▼

GraphQL is a query language for APIs where clients request exactly the data they need — no over-fetching or under-fetching. Unlike REST, it uses a single /graphql endpoint.

const { ApolloServer, gql } = require('apollo-server-express');

const typeDefs = gql`
  type User { id: ID!, name: String! }
  type Query { user(id: ID!): User }
`;

const resolvers = {
  Query: { user: (_, { id }) => getUserById(id) }
};

const server = new ApolloServer({ typeDefs, resolvers });

EQ1 What is Node.js? ▼

Node.js is an open-source, cross-platform JavaScript runtime environment built on Chrome's V8 engine. It allows developers to run JavaScript on the server side, enabling backend development in the same language as frontend.

EQ2 What is Express.js? ▼

Express.js is a fast, minimal, and unopinionated web framework for Node.js. It simplifies routing, middleware integration, and HTTP utilities. It's the most widely used Node.js web framework and forms the "E" in the MEAN/MERN stack.

EQ3 What is the combined use of Node.js and Express.js? ▼

Node.js provides the runtime, Express.js provides the structure. Together they allow you to build production-ready REST APIs, web servers, and microservices quickly.

const express = require('express');
const app = express();
app.use(express.json());

app.get('/api/users', async (req, res) => {
  const users = await User.find();
  res.json(users);
});

app.listen(3000);
EQ4 What is the use of CLI in Node.js? ▼

The Node.js CLI (Command Line Interface) lets you run Node scripts, manage packages, and interact with the REPL.

node app.js          # Run a script
node                 # Open interactive REPL
node -e "console.log(1+1)"  # Execute inline code
node --version       # Check installed version

NPM CLI extends this with package management. You can also build custom CLI tools using packages like commander or yargs.

EQ5 What are the common languages that support Node.js? ▼

Node.js primarily runs JavaScript, but it also supports:

  • TypeScript — via ts-node or compilation to JS
  • CoffeeScript — compiles to JS
  • Dart — via dart2js
  • ClojureScript — compiles to JS

Essentially, any language that compiles to JavaScript can run on Node.js. TypeScript is by far the most popular choice in production today.

EQ6 What is JavaScript engine? ▼

A JavaScript engine is a program that executes JavaScript code. It parses, compiles, and runs JS.

  • V8 — Used by Node.js and Chrome (Google, open-source)
  • SpiderMonkey — Firefox (Mozilla)
  • JavaScriptCore (JSC) — Safari (Apple)
  • Chakra — Old Edge (Microsoft)

V8 compiles JS directly to native machine code using JIT (Just-In-Time) compilation, making it extremely fast.

$.post("/process", {name:'john'}, function(data){ });

// Server code:

  var data = '';
  if(req.xhr){ data += 'One'; }
  if(req.body.name){ data += 'Two'; }
  res.send(data);
});

Answer: "OneTwo"

  • req.xhr is true when the request is made via jQuery's $.post (it sets the X-Requested-With: XMLHttpRequest header) → adds "One"
  • req.body.name is 'john' which is truthy → adds "Two"
  • Final response: "OneTwo"
