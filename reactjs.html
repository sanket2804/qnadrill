🟢 Fresher Questions — Q1 to Q15

Core React fundamentals expected from entry-level candidates

Core Concepts
Q1 What is React.js? Why is it used? ▼

React is an open-source JavaScript library (not a framework) created by Facebook/Meta for building user interfaces — specifically for single-page applications (SPAs). It lets you build reusable UI components and efficiently updates the DOM using a Virtual DOM.

  • Component-based architecture
  • Declarative UI — describe what the UI should look like, React handles how
  • Fast rendering via Virtual DOM diffing
  • Huge ecosystem (React Router, Redux, Next.js)
Interview tip: React is a library, not a full framework like Angular. It only handles the View layer of MVC.
Q2 What is JSX? ▼

JSX (JavaScript XML) is a syntax extension that lets you write HTML-like code inside JavaScript. It is not valid JS — Babel compiles it to React.createElement() calls.

// JSX
const element = <h1 className="title">Hello World</h1>;

// What Babel compiles it to:
const element = React.createElement('h1', { className: 'title' }, 'Hello World');
JSX requires className instead of class, and htmlFor instead of for, since these are reserved JS words.
Q3 What is the Virtual DOM? How does it work? ▼

The Virtual DOM is a lightweight in-memory copy of the real DOM. When state changes, React re-renders the Virtual DOM, diffs it against the previous version (reconciliation), and only updates the changed parts in the real DOM — making updates fast and efficient.

  • Step 1: State changes → React creates new Virtual DOM
  • Step 2: React diffs old vs new Virtual DOM (diffing algorithm)
  • Step 3: Only changed nodes are updated in the real DOM (patching)
Direct DOM manipulation is slow. The Virtual DOM batches and minimizes real DOM operations.
Q4 What is a Component in React? ▼

A component is an independent, reusable piece of UI. React apps are built by composing many small components together. There are two types:

// 1. Functional Component (modern, recommended)
function Greeting({ name }) {
  return <h1>Hello, {name}!</h1>;
}

// 2. Class Component (older style)
class Greeting extends React.Component {
  render() {
    return <h1>Hello, {this.props.name}!</h1>;
  }
}
Q5 What are Props in React? ▼

Props (Properties) are read-only inputs passed from a parent component to a child. They allow data to flow downward in the component tree (unidirectional data flow).

// Parent passes props
<Button label="Click Me" color="blue" onClick={handleClick} />

// Child receives props
function Button({ label, color, onClick }) {
  return <button style={{ color }} onClick={onClick}>{label}</button>;
}
Props are immutable — a child cannot modify its own props. Use state for data that changes.
Q6 What is State in React? ▼

State is mutable data managed within a component. When state changes, React automatically re-renders the component to reflect the new data.

import { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>+</button>
    </div>
  );
}
Q7 What is the difference between State and Props? ▼
  • Props: Passed from parent → child. Read-only. Cannot be changed by the child.
  • State: Managed inside the component. Mutable via setState/useState. Triggers re-render on change.

Analogy: Props are like function arguments (given to you), State is like local variables (you manage them).

Q8 What is the React Component Lifecycle? ▼

Every React component goes through 3 phases:

  • Mounting: Component is created and inserted into DOM (componentDidMount / useEffect(fn, []))
  • Updating: State or props change, component re-renders (componentDidUpdate / useEffect(fn, [dep]))
  • Unmounting: Component is removed from DOM (componentWillUnmount / useEffect cleanup)
useEffect(() => {
  console.log('Mounted');         // componentDidMount
  return () => {
    console.log('Unmounted');     // componentWillUnmount
  };
}, []); // [] = run once
Q9 What is the difference between a Functional and Class Component? ▼
  • Functional: Plain JS function, uses Hooks for state/lifecycle, shorter/cleaner code, better performance, modern standard
  • Class: ES6 class extending React.Component, uses this.state and lifecycle methods, more boilerplate
Since React 16.8 (2019), Hooks made functional components fully capable. All new code should use functional components.
Q10 What are Keys in React and why are they important? ▼

Keys are unique identifiers given to list items to help React identify which items have changed, been added, or removed during re-renders. Without keys, React re-renders the entire list.

const users = [{id: 1, name: 'Alice'}, {id: 2, name: 'Bob'}];

<ul>
  {users.map(user => (
    <li key={user.id}>{user.name}</li>  // ✅ Use id
  ))}
</ul>
Avoid: Using array index as key (key={i}) — causes bugs when the list order changes.
Q11 What is an Event in React? How are events handled? ▼

React uses Synthetic Events — a cross-browser wrapper around native browser events. Event names use camelCase and accept a function (not a string).

function Form() {
  const handleSubmit = (e) => {
    e.preventDefault(); // Prevent page reload
    console.log('Submitted');
  };

  return <form onSubmit={handleSubmit}>
    <button type="submit">Submit</button>
  </form>;
}
Q12 What is Conditional Rendering in React? ▼

Conditional rendering means showing different UI based on conditions, just like JS if/else statements.

// 1. Ternary operator (most common)
{isLoggedIn ? <Dashboard /> : <Login />}

// 2. Short-circuit evaluation
{error && <ErrorMessage msg={error} />}

// 3. if/else before return
if (loading) return <Spinner />;
return <Content />;
Q13 What are React Fragments? ▼

A Fragment lets you group multiple elements without adding an extra DOM node. React components must return a single root element — fragments solve this without adding a meaningless <div>.

// Without fragment — adds unnecessary div
return <div><h1>Title</h1><p>Body</p></div>;

// With fragment — clean DOM
return (
  <>
    <h1>Title</h1>
    <p>Body</p>
  </>
);
Q14 What is Create React App (CRA)? ▼

Create React App was the official CLI tool to scaffold a React project with zero configuration — bundling Webpack, Babel, and ESLint out of the box.

npx create-react-app my-app
cd my-app
npm start
2024 update: CRA is no longer recommended. The React team now recommends Vite for new projects: npm create vite@latest my-app -- --template react
Q15 What is the difference between React and Angular? ▼
  • Type: React is a library (View only); Angular is a full framework (MVC)
  • Language: React uses JavaScript/JSX; Angular uses TypeScript
  • Data binding: React has one-way data flow; Angular has two-way data binding
  • Learning curve: React is simpler to start; Angular has a steeper learning curve
  • Size: React is lighter; Angular is heavier but more opinionated (less decision fatigue)

🔵 Hooks & State Questions — Q16 to Q30

React Hooks — the modern way to manage state and side effects in functional components

Q16 What are React Hooks? Why were they introduced? ▼

Hooks are functions that let you "hook into" React state and lifecycle features from functional components. Introduced in React 16.8 to eliminate the need for class components.

Why introduced:

  • Class components had complex, confusing this binding
  • Logic was scattered across lifecycle methods
  • Hard to reuse stateful logic between components

Common hooks: useState, useEffect, useContext, useRef, useMemo, useCallback, useReducer

Q17 Explain useState hook with an example. ▼

useState adds state to a functional component. It returns an array: the current value and a setter function.

import { useState } from 'react';

function Toggle() {
  const [isOn, setIsOn] = useState(false);

  return (
    <button onClick={() => setIsOn(prev => !prev)}>
      {isOn ? 'ON' : 'OFF'}
    </button>
  );
}
Always use the functional updater form setCount(prev => prev + 1) when new state depends on old state.
Q18 Explain useEffect hook with an example. ▼

useEffect handles side effects (API calls, subscriptions, DOM manipulation). It runs after every render by default, or when dependencies change.

import { useState, useEffect } from 'react';

function UserProfile({ userId }) {
  const [user, setUser] = useState(null);

  useEffect(() => {
    fetch(`/api/users/${userId}`)
      .then(r => r.json())
      .then(setUser);

    return () => { /* cleanup on unmount */ };
  }, [userId]); // Re-runs when userId changes

  return <div>{user?.name}</div>;
}
Q19 What is useContext and when should you use it? ▼

useContext lets you read a Context value without prop drilling. Use it for global data like theme, auth, language settings.

const ThemeContext = React.createContext('light');

// Provider at top level
<ThemeContext.Provider value="dark">
  <App />
</ThemeContext.Provider>

// Consumer anywhere in the tree
function Button() {
  const theme = useContext(ThemeContext);
  return <button className={theme}>Click</button>;
}
Q20 What is useRef and what are its use cases? ▼

useRef returns a mutable ref object whose .current property persists across renders without triggering re-renders.

  • Accessing DOM elements directly (focus, scroll, measure)
  • Storing mutable values that don't need to trigger re-render (timers, previous values)
function TextInput() {
  const inputRef = useRef(null);

  const focusInput = () => inputRef.current.focus();

  return (
    <>
      <input ref={inputRef} />
      <button onClick={focusInput}>Focus</button>
    </>
  );
}
Q21 What is useReducer and when should you use it over useState? ▼

useReducer is an alternative to useState for complex state logic. It uses a reducer function (like Redux) — perfect when next state depends on previous state or multiple sub-values are involved.

const reducer = (state, action) => {
  switch (action.type) {
    case 'increment': return { count: state.count + 1 };
    case 'reset': return { count: 0 };
    default: return state;
  }
};

function Counter() {
  const [state, dispatch] = useReducer(reducer, { count: 0 });
  return (
    <button onClick={() => dispatch({ type: 'increment' })}>
      {state.count}
    </button>
  );
}
Q22 What is useMemo and when should you use it? ▼

useMemo memoizes the result of an expensive calculation so it isn't recomputed on every render — only when dependencies change.

const expensiveResult = useMemo(() => {
  return heavyComputation(data); // Only recalculates when data changes
}, [data]);
Don't overuse useMemo — it has overhead itself. Use only for genuinely expensive calculations (filtering large arrays, complex math).
Q23 What is useCallback and how is it different from useMemo? ▼

useCallback memoizes a function so it's not recreated on every render. useMemo memoizes a value.

// useCallback — memoize function reference
const handleClick = useCallback(() => {
  doSomething(id);
}, [id]); // New function only when id changes

// useMemo — memoize computed value
const total = useMemo(() => cart.reduce((s, i) => s + i.price, 0), [cart]);
Use useCallback when passing callbacks to child components wrapped in React.memo to prevent unnecessary re-renders.
Q24 What are the Rules of Hooks? ▼

React enforces two rules for Hooks:

  • Only call Hooks at the top level — never inside loops, conditions, or nested functions. This ensures hooks are called in the same order every render.
  • Only call Hooks from React functions — functional components or custom hooks. Never from plain JS functions.
// ❌ WRONG — conditional hook
if (user) { useEffect(() => {}); }

// ✅ CORRECT — condition inside hook
useEffect(() => { if (user) { ... } }, [user]);
Q25 What is a Custom Hook and how do you create one? ▼

A custom hook is a reusable function that starts with use and can call other hooks. It extracts stateful logic so it can be shared across components.

// Custom hook: useFetch
function useFetch(url) {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    fetch(url).then(r => r.json()).then(d => {
      setData(d); setLoading(false);
    });
  }, [url]);

  return { data, loading };
}

// Usage in any component
const { data, loading } = useFetch('/api/users');
Q26 What is the useLayoutEffect hook? ▼

useLayoutEffect fires synchronously after all DOM mutations but before the browser paints. It's identical to useEffect but runs earlier.

  • Use useEffect for most side effects (API calls, subscriptions)
  • Use useLayoutEffect when you need to read DOM layout and synchronously re-render (measuring element sizes, preventing flicker)
Caution: useLayoutEffect can block visual updates. Prefer useEffect unless you specifically need synchronous DOM reads.
Q27 What is Prop Drilling and how can you avoid it? ▼

Prop drilling is passing data through multiple layers of components that don't need it, just to reach a deeply nested child.

// Problem — Button doesn't use theme, just passes it down
<App theme="dark"> → <Layout theme="dark"> → <Button theme="dark">

Solutions:

  • React Context — for global data (theme, auth)
  • Redux / Zustand — global state management
  • Component composition — pass components as children/props
Q28 What is the Context API in React? ▼

The Context API provides a way to share data globally across the component tree without prop drilling. It has 3 parts: createContext, Provider, and Consumer/useContext.

// 1. Create context
const AuthContext = React.createContext();

// 2. Provide value
function App() {
  const [user, setUser] = useState(null);
  return (
    <AuthContext.Provider value={{ user, setUser }}>
      <Router />
    </AuthContext.Provider>
  );
}

// 3. Consume anywhere
const { user } = useContext(AuthContext);
Q29 What is the difference between useEffect with [], no array, and [dep]? ▼ 
// Runs once after mount (componentDidMount)
useEffect(() => { fetchData(); }, []);

// Runs after EVERY render (use carefully — can cause infinite loops)
useEffect(() => { document.title = count; });

// Runs when 'count' changes (componentDidUpdate for count)
useEffect(() => { saveToStorage(count); }, [count]);
Always include all variables used inside useEffect in the dependency array. The ESLint plugin eslint-plugin-react-hooks will warn you if you miss one.
Q30 What is the useId hook? ▼

Introduced in React 18, useId generates stable unique IDs that are consistent between server and client renders — useful for accessibility attributes like aria-labelledby or linking form labels to inputs.

function FormField({ label }) {
  const id = useId();
  return (
    <>
      <label htmlFor={id}>{label}</label>
      <input id={id} />
    </>
  );
}
// Generates: id="r:0:", "r:1:", etc.

🟡 Intermediate Questions — Q31 to Q45

Practical React patterns and architecture concepts

Q31 What is Lifting State Up in React? ▼

When multiple components need to share the same state, move the state up to their closest common ancestor. The parent holds state and passes it down as props.

function Parent() {
  const [value, setValue] = useState('');
  return (
    <>
      <InputA value={value} onChange={setValue} />
      <DisplayB value={value} /> {/* Shares same state */}
    </>
  );
}
Q32 What is React.memo and when should you use it? ▼

React.memo is a higher-order component that memoizes a functional component — it skips re-rendering if props haven't changed.

const ExpensiveList = React.memo(({ items }) => {
  return <ul>{items.map(i => <li key={i.id}>{i.name}</li>)}</ul>;
});
// Only re-renders when 'items' prop actually changes
Pair with useCallback for handlers and useMemo for derived data passed as props.
Q33 What are Higher-Order Components (HOC)? ▼

A HOC is a function that takes a component and returns a new enhanced component. It's a pattern for reusing component logic.

// withAuth HOC — adds auth check to any component
function withAuth(Component) {
  return function AuthenticatedComponent(props) {
    const { user } = useAuth();
    if (!user) return <Redirect to="/login" />;
    return <Component {...props} user={user} />;
  };
}

const ProtectedPage = withAuth(Dashboard);
HOCs have largely been replaced by custom hooks in modern React, but they're still common in older codebases and libraries like Redux's connect().
Q34 What are Render Props? ▼

Render Props is a pattern where a component's prop is a function that returns JSX, letting it share logic without HOCs.

function MouseTracker({ render }) {
  const [pos, setPos] = useState({ x: 0, y: 0 });
  return (
    <div onMouseMove={e => setPos({ x: e.clientX, y: e.clientY })}>
      {render(pos)} // Consumer decides what to render
    </div>
  );
}

<MouseTracker render={pos => <p>X: {pos.x}, Y: {pos.y}</p>} />
Q35 What is an Error Boundary in React? ▼

Error Boundaries are class components that catch JavaScript errors in their child component tree and display a fallback UI instead of crashing the whole app.

class ErrorBoundary extends React.Component {
  state = { hasError: false };

  static getDerivedStateFromError() {
    return { hasError: true };
  }

  render() {
    if (this.state.hasError) return <h2>Something went wrong.</h2>;
    return this.props.children;
  }
}
// Usage
<ErrorBoundary><MyComponent /></ErrorBoundary>
Error Boundaries don't catch errors in event handlers, async code, or SSR — only in render and lifecycle methods.
Q36 What is Lazy Loading and React.lazy()? ▼

Lazy loading defers loading a component until it's actually needed, reducing the initial bundle size and improving load time.

import { lazy, Suspense } from 'react';

const HeavyChart = lazy(() => import('./HeavyChart'));

function Dashboard() {
  return (
    <Suspense fallback={<Spinner />}>
      <HeavyChart />
    </Suspense>
  );
}
Q37 What is Reconciliation in React? ▼

Reconciliation is React's algorithm for updating the DOM efficiently. When state/props change, React compares the new Virtual DOM with the previous one (diffing) and only applies the minimum changes needed to the real DOM.

  • Elements of different types → React destroys old tree, builds new one
  • Elements of same type → React updates only changed attributes
  • Lists → React uses key prop to match items
Q38 What is Controlled vs Uncontrolled Component? ▼
  • Controlled: Form data is handled by React state. The input value is always driven by state.
  • Uncontrolled: Form data is handled by the DOM itself. You read values using refs.
// Controlled
<input value={name} onChange={e => setName(e.target.value)} />

// Uncontrolled
const ref = useRef();
<input ref={ref} defaultValue="Alice" />
// Access via: ref.current.value
Controlled components are recommended for most forms — they give you full control for validation and conditional logic.
Q39 What is a Portal in React? ▼

Portals render a child component into a different DOM node than the parent's DOM hierarchy — useful for modals, tooltips, and dropdowns that need to escape parent CSS overflow or z-index constraints.

import { createPortal } from 'react-dom';

function Modal({ children }) {
  return createPortal(
    <div className="modal">{children}</div>,
    document.getElementById('modal-root') // Renders here, not in parent
  );
}
Q40 What is Strict Mode in React? ▼

React.StrictMode is a development-only tool that helps detect potential problems by intentionally double-invoking certain functions and warning about deprecated APIs.

import { StrictMode } from 'react';
<StrictMode>
  <App />
</StrictMode>

It detects: components with unsafe lifecycle methods, unexpected side effects (by calling useEffect twice), legacy string refs, and deprecated findDOMNode usage.

Q41 What is the difference between SPA and MPA? Where does React fit? ▼
  • SPA (Single Page Application): One HTML page loaded once; navigation happens via JS without full page reloads. Fast, app-like feel. React, Angular, Vue build SPAs.
  • MPA (Multi Page Application): Each route is a separate HTML page from the server. Traditional PHP/Rails sites. Better SEO by default.

React is primarily used for SPAs, but with Next.js (SSR/SSG), React can also power MPAs and hybrid apps with excellent SEO.

Q42 What is Server-Side Rendering (SSR) in React? ▼

SSR renders React components to HTML on the server before sending to the browser. The user sees content faster (no blank screen), and search engines can index it immediately.

  • CSR (default React): Browser downloads JS → runs React → renders UI
  • SSR (Next.js): Server renders HTML → browser receives ready HTML → React "hydrates" it
Use Next.js for SSR with React. It handles all the complexity with getServerSideProps or the App Router's Server Components.
Q43 What is the difference between React.Component and React.PureComponent? ▼
  • React.Component: Re-renders every time setState is called, even if the state/props didn't actually change
  • React.PureComponent: Implements shouldComponentUpdate with a shallow comparison — skips re-render if props/state are the same

The functional component equivalent of PureComponent is React.memo().

Shallow comparison only checks the top-level reference. Nested object mutations won't trigger updates — always create new objects/arrays when updating state.
Q44 How do you fetch data in React? ▼ 
// 1. useEffect + fetch (basic)
const [data, setData] = useState(null);
useEffect(() => {
  fetch('/api/data')
    .then(r => r.json())
    .then(setData);
}, []);

// 2. Custom hook (reusable)
const { data, loading, error } = useFetch('/api/data');

// 3. React Query (recommended for production)
const { data } = useQuery({ queryKey: ['data'], queryFn: fetchData });
For production apps, use TanStack Query (React Query) — it handles caching, refetching, loading/error states automatically.
Q45 What is Synthetic Event in React? ▼

React wraps native browser events in a Synthetic Event — a cross-browser normalized wrapper that ensures consistent behavior across all browsers. It pools events for performance.

function handleClick(e) {
  console.log(e.type);        // 'click'
  console.log(e.target);      // The DOM element clicked
  e.preventDefault();         // Works cross-browser
  e.stopPropagation();        // Stops event bubbling
}

🔴 Advanced Questions — Q46 to Q55

Deep-dive React internals and architecture for senior developers

Q46 What is React Fiber? How is it different from the old reconciler? ▼

React Fiber (React 16+) is a complete rewrite of React's core reconciliation algorithm. The old "Stack" reconciler was synchronous — it had to finish rendering in one go, blocking the browser.

Fiber introduced incremental rendering: it can split rendering work into chunks, pause and resume it, prioritize updates, and abort work if no longer needed.

  • High-priority updates (user input) interrupt low-priority ones (data loading)
  • Enables Concurrent Mode, Suspense, and Time Slicing
Q47 What is React Concurrent Mode? ▼

Concurrent Mode (React 18) allows React to prepare multiple versions of the UI simultaneously. It makes apps feel more responsive by keeping the UI interactive during heavy renders.

  • startTransition: Mark non-urgent updates to keep UI responsive
  • useTransition: Track pending state of transitions
  • useDeferredValue: Defer re-rendering of a slow part of the UI
import { startTransition, useTransition } from 'react';

const [isPending, startTransition] = useTransition();

startTransition(() => {
  setSearchResults(filterLargeList(query)); // Non-urgent
});
Q48 What are React Server Components (RSC)? ▼

Introduced in React 18 and heavily used in Next.js 13+ App Router. Server Components run exclusively on the server — they can directly access databases, file system, and APIs without sending JS to the client.

  • Server Components: No useState, no useEffect, no browser APIs. Zero JS bundle cost.
  • Client Components: "use client" directive. Full interactivity with hooks.
// ServerPage.jsx — runs on server, no JS sent to browser
async function ProductPage({ id }) {
  const product = await db.products.findById(id); // Direct DB call
  return <ProductCard product={product} />;
}
Q49 What is Suspense in React and how does it work? ▼

Suspense lets you declaratively specify a loading state while waiting for async operations (lazy-loaded components, data fetching with React Query or RSC).

<Suspense fallback={<Skeleton />}>
  <SlowDataComponent />  // Suspended until data ready
</Suspense>

React 18 expanded Suspense for data fetching — components can "suspend" by throwing a Promise, and the nearest <Suspense> catches it and shows the fallback.

Q50 How does React handle batching of state updates? ▼

React batches multiple state updates into a single re-render for performance. In React 18, Automatic Batching extends this to all async contexts (setTimeout, fetch callbacks, etc.) — not just event handlers.

// React 18: Both setters batch into ONE re-render
setTimeout(() => {
  setCount(c => c + 1);
  setFlag(f => !f);
  // Only ONE re-render total
}, 1000);

// Opt out with flushSync if needed
import { flushSync } from 'react-dom';
flushSync(() => setCount(c => c + 1)); // Forces immediate re-render
Q51 What is forwardRef and when do you use it? ▼

forwardRef lets parent components pass a ref down to a child DOM element. By default, you can't attach a ref to a functional component — forwardRef enables this.

const Input = React.forwardRef((props, ref) => (
  <input ref={ref} {...props} />
));

// Parent can now focus the inner input
const ref = useRef();
<Input ref={ref} />
ref.current.focus();
Q52 What is useImperativeHandle? ▼

useImperativeHandle customizes what the parent gets when they attach a ref to your component — instead of exposing the entire DOM node, you expose a specific API.

const VideoPlayer = forwardRef((props, ref) => {
  const videoRef = useRef();

  useImperativeHandle(ref, () => ({
    play: () => videoRef.current.play(),
    pause: () => videoRef.current.pause(),
  }));

  return <video ref={videoRef} src={props.src} />;
});
Q53 What is the difference between SSR, SSG, and ISR? ▼
  • SSR (Server-Side Rendering): Page is rendered on the server on every request. Fresh data always. Slower TTFB. Use for highly dynamic content (e.g., user dashboards).
  • SSG (Static Site Generation): Page is pre-rendered at build time. Super fast. Use for content that rarely changes (blogs, docs).
  • ISR (Incremental Static Regeneration): Next.js feature — page is statically generated but auto-revalidated in the background at set intervals. Best of both worlds.
ISR is the sweet spot for most content sites — fast like SSG, but data stays fresh.
Q54 What is the React component composition pattern? ▼

Composition means building complex UIs by combining smaller, focused components — favored over inheritance. The children prop and slot patterns enable flexible composition.

function Card({ header, children, footer }) {
  return (
    <div className="card">
      <div className="card-header">{header}</div>
      <div className="card-body">{children}</div>
      <div className="card-footer">{footer}</div>
    </div>
  );
}
Q55 How do you handle forms in React at scale? ▼

For simple forms, use controlled components with useState. For complex forms at scale, use dedicated libraries:

  • React Hook Form — minimal re-renders, excellent performance, built-in validation
  • Formik — full-featured, integrates with Yup for schema validation
  • Zod — TypeScript-first schema validation, pairs with React Hook Form
// React Hook Form example
const { register, handleSubmit, formState: { errors } } = useForm();
<input {...register('email', { required: true, pattern: /^\S+@\S+$/ })} />

⚡ Performance Optimization — Q56 to Q62

Techniques to make React apps fast and efficient

Q56 What causes unnecessary re-renders in React and how do you fix them? ▼

Causes:

  • Parent re-renders → all children re-render by default
  • New object/array/function references as props on each render
  • Context value changing triggers all consumers to re-render

Fixes:

  • Wrap components with React.memo
  • Stabilize function props with useCallback
  • Stabilize object/array props with useMemo
  • Split contexts so only relevant consumers update
Q57 What is Code Splitting in React? ▼

Code splitting breaks your JS bundle into smaller chunks loaded on demand — reducing initial load time. React supports it via React.lazy() and dynamic import().

// Split by route — only load page when navigating there
const Settings = lazy(() => import('./pages/Settings'));
const Analytics = lazy(() => import('./pages/Analytics'));

<Routes>
  <Route path="/settings" element={<Suspense fallback={<Spinner />}><Settings /></Suspense>} />
</Routes>
Q58 How does virtualization improve list performance? ▼

Rendering 10,000 list items creates 10,000 DOM nodes — very slow. Virtualization (or windowing) only renders items visible in the viewport, dramatically reducing DOM nodes.

import { FixedSizeList } from 'react-window';

<FixedSizeList
  height={600}
  itemCount={10000}
  itemSize={50}
>
  {({ index, style }) => <div style={style}>Row {index}</div>}
</FixedSizeList>

Libraries: react-window (lightweight), react-virtual (TanStack, flexible).

Q59 What is the React Profiler and how do you use it? ▼

The React Profiler (in React DevTools) measures how often components render and the "cost" of each render. It helps find performance bottlenecks.

You can also use the Profiler component programmatically:

<Profiler id="Navigation" onRender={
  (id, phase, actualDuration) => {
    console.log(`${id} took ${actualDuration}ms`);
  }
}>
  <Navigation />
</Profiler>
Q60 How do you optimize images in a React application? ▼
  • Use lazy loading: <img loading="lazy" /> or the Intersection Observer API
  • Use next/image in Next.js — automatic optimization, WebP conversion, responsive sizes
  • Use modern formats: WebP / AVIF over PNG/JPEG
  • Use a CDN (Cloudinary, imgix) for automatic resizing and delivery

Q61 What is Tree Shaking and how does it apply to React? ▼

Tree shaking is a bundler optimization (Webpack, Vite) that removes unused code from the final bundle. It works with ES Module syntax (import/export).

// ❌ Imports entire lodash (70KB)
import _ from 'lodash';

// ✅ Tree-shakeable — only imports debounce
import debounce from 'lodash/debounce';

Ensure libraries you use support tree shaking (check for "sideEffects": false in their package.json).

Q62 What is debouncing and throttling in React and when do you use each? ▼
  • Debouncing: Delays execution until after a pause in calls. Use for search inputs — fire API only after user stops typing.
  • Throttling: Limits execution to once per time interval. Use for scroll/resize handlers.
import { useMemo } from 'react';
import debounce from 'lodash/debounce';

const debouncedSearch = useMemo(
  () => debounce(query => fetchResults(query), 300),
  []
);

<input onChange={e => debouncedSearch(e.target.value)} />

Q63 What is React Router and how does it work? ▼

React Router is the standard routing library for React SPAs. It maps URL paths to components without actual page reloads, using the browser's History API.

import { BrowserRouter, Routes, Route, Link } from 'react-router-dom';

<BrowserRouter>
  <nav>
    <Link to="/">Home</Link>
    <Link to="/about">About</Link>
  </nav>
  <Routes>
    <Route path="/" element={<Home />} />
    <Route path="/about" element={<About />} />
    <Route path="/user/:id" element={<UserProfile />} />
  </Routes>
</BrowserRouter>
Q64 What is the difference between useNavigate, useParams, and useLocation? ▼ 
// useNavigate — programmatic navigation
const navigate = useNavigate();
navigate('/dashboard');
navigate(-1); // Go back

// useParams — read URL parameters
// Route: /users/:id
const { id } = useParams(); // id = '42' from /users/42

// useLocation — read current URL info
const location = useLocation();
console.log(location.pathname); // '/users/42'
console.log(location.search);   // '?tab=settings'
Q65 What is Protected Route and how do you implement it? ▼

A Protected Route redirects unauthenticated users to the login page instead of rendering the protected component.

function ProtectedRoute({ children }) {
  const { user } = useAuth();
  if (!user) return <Navigate to="/login" replace />;
  return children;
}

// Usage
<Route path="/dashboard" element={
  <ProtectedRoute><Dashboard /></ProtectedRoute>
} />
Q66 What is Redux and what problem does it solve? ▼

Redux is a predictable state management library. It solves the problem of sharing complex state across many components by storing all app state in a single centralized store.

  • Store: Single source of truth for all state
  • Action: Plain object describing what happened { type: 'ADD_TODO', payload: 'Buy milk' }
  • Reducer: Pure function that computes new state from action
  • Dispatch: Sends an action to the store
For most apps, React Context + useReducer or Zustand is simpler than Redux. Use Redux for very large apps with complex state needs.
Q67 What is Redux Toolkit (RTK) and how is it different from plain Redux? ▼

Redux Toolkit is the official, opinionated way to write Redux — it eliminates boilerplate with createSlice, includes Immer for immutable updates, and adds RTK Query for data fetching.

// RTK createSlice — replaces action creators + reducer
const counterSlice = createSlice({
  name: 'counter',
  initialState: { value: 0 },
  reducers: {
    increment: state => { state.value += 1; }, // Immer handles immutability
    reset: state => { state.value = 0; }
  }
});
export const { increment, reset } = counterSlice.actions;
Q68 What is Zustand and how does it compare to Redux? ▼

Zustand is a lightweight state management library with a minimal API — much simpler than Redux, no providers needed, no boilerplate.

import { create } from 'zustand';

const useStore = create(set => ({
  count: 0,
  increment: () => set(state => ({ count: state.count + 1 })),
  reset: () => set({ count: 0 })
}));

// In component — no Provider needed
const { count, increment } = useStore();
Zustand is great for small to medium apps. Redux Toolkit is better for very large teams/apps needing strict conventions and devtools.
Q69 What is the Flux architecture pattern? ▼

Flux is Facebook's application architecture pattern for building client-side web apps. It enforces unidirectional data flow:

Action → Dispatcher → Store → View → Action (repeat)

  • Actions describe what happened
  • Dispatcher routes actions to stores
  • Stores hold state and business logic
  • Views (React components) render based on store state

Redux is the most popular implementation of Flux principles, simplifying the Dispatcher into reducers.

Q70 What is Redux Thunk and Redux Saga? When do you use each? ▼

Both are Redux middleware for handling async operations (API calls, side effects).

  • Redux Thunk: Simple — lets action creators return functions instead of objects. Good for straightforward async logic.
  • Redux Saga: Advanced — uses generator functions for complex async flows (cancellation, racing, polling, retry logic).
// Thunk — simple async action
const fetchUser = (id) => async (dispatch) => {
  const user = await api.getUser(id);
  dispatch({ type: 'SET_USER', payload: user });
};
Use Thunk for simple cases. Use Saga when you need features like takeLatest (cancel in-flight requests), retry, or complex sequencing. RTK Query often replaces both for data fetching.
