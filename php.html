🟢 Basics — Q1 to Q15

Fundamental PHP concepts expected from any candidate

Q1 What is PHP? What are its main features? ▼

PHP (Hypertext Preprocessor) is a server-side scripting language designed for web development. It is embedded in HTML, runs on the server, and sends the output (HTML) to the browser.

  • Open-source and free — large ecosystem and community
  • Cross-platform — runs on Windows, Linux, macOS
  • Loosely typed — variables don't need a declared type
  • Huge standard library — 1000s of built-in functions
  • Database support — MySQL, PostgreSQL, SQLite, Oracle, and more
  • Supports OOP — classes, interfaces, traits, abstract classes
// Basic PHP script embedded in HTML
<?php
  $name = "World";
  echo "Hello, $name!";
?>
Interview tip: PHP powers 78%+ of websites with a known server-side language — including WordPress, Facebook (originally), and Wikipedia.
Q2 What are PHP data types? ▼

PHP supports 8 primitive data types split into three categories:

  • Scalar: int, float, string, bool
  • Compound: array, object
  • Special: null, resource
$age    = 25;           // int
$price  = 9.99;         // float
$name   = "Alice";      // string
$active = true;         // bool
$tags   = ["php", "web"]; // array
$db     = null;          // null

var_dump($age);          // int(25)
Q3 What is the difference between == and === in PHP? ▼
  • == (Loose comparison): Compares values after type coercion — "1" == 1 is true
  • === (Strict comparison): Compares both value AND type — "1" === 1 is false
$a = "1";
$b = 1;

$a == $b;   // true  (type coerced)
$a === $b;  // false (different types)

null == false;   // true
null === false;  // false
0 == "foo";     // false (PHP 8 changed this!)
Best practice: Always use === to avoid subtle bugs from type juggling.
Q4 What are PHP variable scopes? ▼

PHP has four types of variable scope:

  • Local: Declared inside a function — only accessible within it
  • Global: Declared outside functions — use global $var or $GLOBALS['var'] inside functions
  • Static: Local variable that persists across function calls using static
  • Superglobal: Built-in arrays accessible everywhere: $_GET, $_POST, $_SESSION, $_SERVER, etc.
$count = 0; // global

function increment() {
  static $calls = 0; // persists between calls
  global $count;     // access global
  $calls++;
  $count++;
  echo $calls;
}
increment(); // 1
increment(); // 2 (static persists)
Q5 What is the difference between echo and print in PHP? ▼
  • echo: Outputs one or more strings. No return value. Slightly faster. Can take multiple arguments.
  • print: Outputs one string. Always returns 1 — can be used in expressions.
echo "Hello", " World"; // multiple args OK
print "Hello";           // returns 1
$x = print "test";      // $x = 1
In practice, echo is used almost universally. print is rarely used.
Q6 What are PHP Superglobals? ▼

Superglobals are built-in variables accessible from anywhere in a PHP script — inside functions, classes, or global scope — without needing the global keyword.

  • $_GET — query string parameters from URL
  • $_POST — form data submitted via POST
  • $_REQUEST — combined GET, POST, COOKIE
  • $_SESSION — session data
  • $_COOKIE — browser cookies
  • $_FILES — uploaded file data
  • $_SERVER — server and execution environment info
  • $_ENV — environment variables
  • $GLOBALS — all global variables
Q7 What is the difference between GET and POST methods? ▼
  • GET: Data sent in URL query string. Visible in browser. Cached and bookmarkable. Limited to ~2000 chars. Use for fetching/reading data.
  • POST: Data sent in request body. Not visible in URL. Not cached. No size limit. Use for submitting/changing data.
// GET: /search?q=php&page=2
$query = $_GET['q'];

// POST: form submission
$email = $_POST['email'];
Never trust data from $_GET or $_POST directly. Always sanitize and validate.
Q8 What are PHP Sessions and Cookies? What is the difference? ▼
  • Session: Stored on the server. More secure. Expires when browser closes (by default). Identified by a session ID cookie.
  • Cookie: Stored in user's browser. Can persist long-term. Less secure — user can modify them.
// Session
session_start();
$_SESSION['user_id'] = 42;
echo $_SESSION['user_id']; // 42

// Cookie
setcookie('theme', 'dark', time() + 86400); // expires in 1 day
echo $_COOKIE['theme']; // 'dark'
Q9 What is type juggling in PHP? ▼

PHP automatically converts types when an operation requires a different type — this is called type juggling (or type coercion). It happens implicitly without casting.

$result = "5" + 3;      // int 8 (string "5" → int)
$str    = "10 cats" + 5; // int 15 (leading number extracted)
$flag   = "" + 0;        // int 0 (empty string → 0)

// Explicit casting
$a = (int) "42abc";  // int 42
$b = (bool) "";      // false
$c = (string) 3.14; // "3.14"
PHP 8 made some type juggling stricter — e.g., 0 == "foo" is now false (was true in PHP 7).
Q10 What is the difference between include, require, include_once, and require_once? ▼
  • include: Includes a file. On failure, emits a warning and continues execution.
  • require: Includes a file. On failure, emits a fatal error and stops execution.
  • include_once: Same as include but checks if already included — prevents duplicate declarations.
  • require_once: Same as require but checks if already included.
require_once 'config.php';  // Critical — stop if missing
include 'sidebar.php';      // Optional — continue if missing
Use require_once for critical files (config, database). Use include for optional templates.
Q11 What are PHP constants? How are they different from variables? ▼

Constants are identifiers with a fixed value that cannot change during execution. Unlike variables, they have no $ prefix and are global by default.

// Defining constants
define('MAX_SIZE', 1024);
const VERSION = '2.0';

echo MAX_SIZE; // 1024
echo VERSION;  // 2.0

// PHP built-in constants
echo PHP_VERSION;   // e.g. 8.2.0
echo PHP_INT_MAX;   // 9223372036854775807
echo __FILE__;      // current file path
echo __LINE__;      // current line number
Q12 What are the different types of errors in PHP? ▼
  • E_ERROR: Fatal runtime error — script stops (e.g., calling undefined function)
  • E_WARNING: Non-fatal runtime warning — script continues (e.g., include missing file)
  • E_NOTICE: Minor runtime notice (e.g., accessing undefined variable)
  • E_PARSE: Compile-time parse error — script cannot run
  • E_DEPRECATED: Using deprecated functions
  • E_STRICT: Code compatibility suggestions (merged into E_NOTICE in PHP 8)
// Error reporting configuration
error_reporting(E_ALL);           // Show all errors
ini_set('display_errors', '1');  // Show in browser (dev only)

// Custom error handler
set_error_handler(function($errno, $errstr) {
  error_log("Error $errno: $errstr");
});
Q13 How does PHP handle exceptions? ▼

PHP uses try-catch-finally blocks for exception handling. You can throw built-in exceptions or create custom ones by extending the Exception class.

class ValidationException extends Exception {}

function validateAge($age) {
  if ($age < 0) {
    throw new ValidationException("Age cannot be negative");
  }
}

try {
  validateAge(-1);
} catch (ValidationException $e) {
  echo "Validation: " . $e->getMessage();
} catch (Exception $e) {
  echo "General: " . $e->getMessage();
} finally {
  echo "Always runs";
}
Q14 What is the difference between single-quoted and double-quoted strings? ▼
  • Single quotes ('): Literal string — no variable interpolation, no escape sequences except \' and \\. Slightly faster.
  • Double quotes ("): Variables and escape sequences are parsed (\n, \t, \$, etc.)
$name = "Alice";

echo 'Hello $name\n';    // Hello $name\n  (literal)
echo "Hello $name\n";    // Hello Alice    (interpolated)
echo "Hello {$name}!";   // Hello Alice!   (curly braces)
Q15 What is PHP's null coalescing operator and spaceship operator? ▼

Null coalescing (??): Returns the left operand if it exists and is not null; otherwise returns the right. Added in PHP 7.

Spaceship (<=>): Compares two values. Returns -1, 0, or 1. Useful for sorting. Added in PHP 7.

// Null coalescing
$user = $_GET['user'] ?? 'Guest';  // 'Guest' if not set
$config['timeout'] ??= 30;          // assign if null (PHP 7.4+)

// Spaceship operator
echo 1 <=> 2;  // -1
echo 2 <=> 2;  //  0
echo 3 <=> 2;  //  1

usort($arr, fn($a, $b) => $a['age'] <=> $b['age']);

🔵 Object-Oriented PHP — Q16 to Q30

OOP concepts, design patterns, and class-based features

Q16 What are the four pillars of OOP in PHP? ▼
  • Encapsulation: Bundling data (properties) and methods together inside a class, and controlling access via public, protected, private.
  • Inheritance: A child class extends a parent class, inheriting its properties and methods using extends.
  • Polymorphism: Different classes implementing the same interface/method, each behaving differently. Achieved via method overriding and interfaces.
  • Abstraction: Hiding implementation details and exposing only what's necessary — via abstract classes and interfaces.
Q17 What is the difference between abstract class and interface? ▼
  • Abstract class: Can have both abstract methods (no body) and concrete methods (with body). Can have properties. A class can extend only ONE abstract class.
  • Interface: All methods are abstract by default (no body). No properties (only constants). A class can implement MULTIPLE interfaces.
interface Drawable {
  public function draw(): void;
}

abstract class Shape {
  abstract public function area(): float;
  public function describe() { echo "I am a shape"; }
}

class Circle extends Shape implements Drawable {
  public function area(): float { return pi() * 5 ** 2; }
  public function draw(): void { echo "Drawing circle"; }
}
Q18 What are access modifiers in PHP? ▼
  • public: Accessible from anywhere — inside the class, subclasses, and outside.
  • protected: Accessible only inside the class and its subclasses. Not from outside.
  • private: Accessible only inside the defining class. Not even subclasses.
class BankAccount {
  public    $owner;
  protected $balance = 0;
  private   $pin;

  public function deposit($amt) {
    $this->balance += $amt; // OK
  }
  private function verifyPin($p) {
    return $p === $this->pin;
  }
}
Q19 What is a Trait in PHP? Why is it used? ▼

A Trait is a mechanism for code reuse in single-inheritance languages like PHP. It lets you include methods in multiple classes without inheritance. Traits solve the "diamond problem" of multiple inheritance.

trait Timestampable {
  public function createdAt(): string {
    return date('Y-m-d H:i:s');
  }
}

trait Loggable {
  public function log($msg) { error_log($msg); }
}

class User {
  use Timestampable, Loggable; // Multiple traits
}

$u = new User();
echo $u->createdAt();
Q20 What is the difference between $this, self::, static::, and parent::? ▼
  • $this: Refers to the current object instance. Used for non-static properties/methods.
  • self::: Refers to the class where it is written (compile-time). Does NOT support late static binding.
  • static::: Refers to the class that was called at runtime — supports Late Static Binding (PHP 5.3+).
  • parent::: Refers to the parent class — used to call overridden parent methods.
class Base {
  public static function create() {
    return new static(); // Late Static Binding
  }
}
class Child extends Base {}

$obj = Child::create(); // Returns Child, not Base
Q21 What is method overriding and method overloading in PHP? ▼
  • Method Overriding: Child class re-defines a parent class method with the same name. PHP supports this natively.
  • Method Overloading: PHP doesn't support traditional overloading. Instead, it uses magic methods __call() and __callStatic() to intercept calls to undefined methods.
// Overriding
class Animal {
  public function speak() { echo "..."; }
}
class Dog extends Animal {
  public function speak() { echo "Woof!"; } // overrides
}

// "Overloading" via __call
public function __call($name, $args) {
  echo "Called $name with " . count($args) . " args";
}
Q22 What are PHP Magic Methods? ▼

Magic methods are special methods that start with __ (double underscore) and are automatically called in certain situations.

  • __construct() — called when object is created
  • __destruct() — called when object is destroyed
  • __get($name) — called when accessing inaccessible property
  • __set($name, $val) — called when setting inaccessible property
  • __isset($name) — called with isset() on inaccessible property
  • __toString() — called when object used as string
  • __invoke() — called when object used as function
  • __clone() — called when object is cloned
  • __call($name, $args) — called when calling inaccessible method
class Money {
  public function __construct(private float $amount) {}
  public function __toString(): string {
    return "$" . number_format($this->amount, 2);
  }
}
echo new Money(9.5); // $9.50
Q23 What is the Singleton design pattern in PHP? ▼

The Singleton pattern ensures only one instance of a class is created throughout the application's lifecycle. Commonly used for database connections.

class Database {
  private static ?Database $instance = null;
  private PDO $pdo;

  private function __construct() {
    $this->pdo = new PDO('mysql:host=localhost;dbname=app', 'user', 'pass');
  }

  public static function getInstance(): Database {
    if (self::$instance === null) {
      self::$instance = new self();
    }
    return self::$instance;
  }

  private function __clone() {} // Prevent cloning
}

$db = Database::getInstance();
Q24 What is a Constructor and Destructor in PHP? ▼

Constructor (__construct): Automatically called when a new object is instantiated. Used to initialize properties.

Destructor (__destruct): Automatically called when an object goes out of scope or the script ends. Used for cleanup (closing DB connections, files).

class FileLogger {
  private $handle;

  public function __construct(string $path) {
    $this->handle = fopen($path, 'a');
  }

  public function __destruct() {
    fclose($this->handle); // auto cleanup
  }
}
Q25 What is Late Static Binding (LSB) in PHP? ▼

LSB allows you to reference the class that was called at runtime (not the class where the method was defined) using static:: instead of self::.

class ParentClass {
  public static function create() {
    return new static(); // LSB — runtime class
    // new self() would always return ParentClass
  }
  public static function className() {
    return static::class; // LSB
  }
}
class ChildClass extends ParentClass {}

ChildClass::create();    // Returns ChildClass instance
ChildClass::className(); // Returns "ChildClass"
Q26 What is the difference between shallow copy and deep copy (clone)? ▼

In PHP, objects are assigned by reference by default. Using clone creates a shallow copy — primitive properties are copied, but object properties still point to the same object. Override __clone() for deep copy.

class Cart {
  public Address $address;

  public function __clone() {
    $this->address = clone $this->address; // deep copy
  }
}

$cart1 = new Cart();
$cart2 = clone $cart1; // independent copy
Q27 What is Dependency Injection in PHP? ▼

Dependency Injection (DI) is a design pattern where a class receives its dependencies from outside rather than creating them internally. Makes code testable, flexible, and loosely coupled.

// Without DI — tightly coupled, hard to test
class OrderService {
  private $mailer;
  public function __construct() {
    $this->mailer = new SmtpMailer(); // hardcoded
  }
}

// With DI — loosely coupled, testable
class OrderService {
  public function __construct(private MailerInterface $mailer) {}
}

$service = new OrderService(new SmtpMailer());
// In tests: new OrderService(new MockMailer())
Q28 What are PHP Enums? (PHP 8.1+) ▼

Enums (Enumerations) introduced in PHP 8.1 allow defining a type with a fixed set of named values — replacing class constants and reducing bugs from magic strings/integers.

// Pure enum
enum Status {
  case Active;
  case Inactive;
  case Pending;
}

// Backed enum (with values)
enum Color: string {
  case Red   = 'red';
  case Green = 'green';
  case Blue  = 'blue';
}

$status = Status::Active;
$color  = Color::from('red'); // Color::Red
Q29 What is an anonymous class in PHP? ▼

Anonymous classes (PHP 7+) are classes without a name, defined inline where they're used. Useful for one-off implementations — especially in testing (mocking).

interface Logger {
  public function log(string $msg): void;
}

// Anonymous class implementing an interface
$logger = new class implements Logger {
  public function log(string $msg): void {
    echo "[LOG] $msg";
  }
};

$logger->log("Server started");
Q30 What is PHP Namespacing? ▼

Namespaces prevent naming conflicts between classes, functions, and constants — especially important when combining multiple libraries. They work like directories for code.

// File: App/Models/User.php
namespace App\Models;

class User { ... }

// File: App/Http/Controllers/UserController.php
namespace App\Http\Controllers;

use App\Models\User;
use App\Services\UserService as Service; // alias

class UserController {
  public function index() {
    return User::all();
  }
}

🟡 Functions & Arrays — Q31 to Q45

Built-in functions, array manipulation, closures, and string handling

Q31 What is the difference between array_map, array_filter, and array_reduce? ▼
  • array_map: Applies a callback to each element, returning a new array of the same size.
  • array_filter: Keeps only elements where callback returns true. Keys are preserved.
  • array_reduce: Reduces array to a single value by accumulating with a callback.
$nums = [1, 2, 3, 4, 5];

// map: double each
$doubled = array_map(fn($n) => $n * 2, $nums);
// [2, 4, 6, 8, 10]

// filter: keep evens
$evens = array_filter($nums, fn($n) => $n % 2 === 0);
// [2, 4]

// reduce: sum all
$sum = array_reduce($nums, fn($carry, $item) => $carry + $item, 0);
// 15
Q32 What are Closures (anonymous functions) in PHP? ▼

A closure is an anonymous function that can capture variables from its surrounding scope using the use keyword. They are objects of the built-in Closure class.

$multiplier = 3;

$multiply = function($n) use ($multiplier) {
  return $n * $multiplier;
};

echo $multiply(5);  // 15

// Arrow function (PHP 7.4+) — auto-captures by value
$multiply = fn($n) => $n * $multiplier;
echo $multiply(5);  // 15
Q33 What is the difference between array_push and the [] operator? ▼
  • array_push($arr, $val): Adds one or more elements to the end of an array. Returns new count. Has function call overhead.
  • $arr[] = $val: Direct language construct. Faster. Preferred for single element addition.
$arr = [1, 2, 3];

array_push($arr, 4, 5);  // [1,2,3,4,5] — multiple at once
$arr[] = 6;              // [1,2,3,4,5,6] — preferred for single
Q34 What is the difference between sort(), asort(), ksort(), and usort()? ▼
  • sort(): Sorts values ascending. Resets numeric keys.
  • asort(): Sorts values ascending. Preserves key association.
  • ksort(): Sorts by keys ascending. Preserves key-value association.
  • usort(): Sorts by a user-defined comparison function.
$people = [
  ['name' => 'Charlie', 'age' => 30],
  ['name' => 'Alice',   'age' => 25],
];

usort($people, fn($a, $b) => $a['age'] <=> $b['age']);
// Alice (25), Charlie (30)
Q35 What is the difference between array_merge and array_combine and array_merge with + operator? ▼
  • array_merge: Merges arrays. String keys from later arrays overwrite earlier. Numeric keys are re-indexed.
  • + operator: Merges arrays. Earlier keys are preserved — later duplicates are ignored.
  • array_combine: Creates an array using one array as keys and another as values.
$a = ['x' => 1, 0 => 'A'];
$b = ['x' => 2, 0 => 'B'];

array_merge($a, $b); // ['x'=>2, 0=>'A', 1=>'B']
$a + $b;            // ['x'=>1, 0=>'A']  (a wins)

array_combine(['a','b'], [1,2]); // ['a'=>1, 'b'=>2]
Q36 What are common PHP string functions? ▼ 
strlen("Hello");             // 5
strtolower("PHP");           // "php"
strtoupper("php");           // "PHP"
str_replace("a","@","abc"); // "@bc"
substr("Hello", 1, 3);       // "ell"
strpos("Hello", "l");        // 2
trim("  hi  ");             // "hi"
explode(",", "a,b,c");      // ["a","b","c"]
implode("-", ["a","b"]);    // "a-b"
str_pad("5", 3, "0", STR_PAD_LEFT); // "005"
sprintf("%.2f", 3.14159);   // "3.14"
str_contains("hello", "ell"); // true (PHP 8+)
Q37 What is pass by value vs pass by reference in PHP? ▼
  • Pass by value (default): A copy of the variable is passed. Changes inside the function don't affect the original.
  • Pass by reference (&): The actual variable is passed. Changes inside the function affect the original.
function addTen($n) { $n += 10; }
function addTenRef(&$n) { $n += 10; }

$x = 5;
addTen($x);    echo $x; // 5  (unchanged)
addTenRef($x); echo $x; // 15 (modified)
Q38 What is a variadic function in PHP? ▼

A variadic function accepts a variable number of arguments using the ... (splat) operator. The arguments are collected into an array.

function sum(int ...$nums): int {
  return array_sum($nums);
}
echo sum(1, 2, 3, 4); // 10

// Splat to unpack array into args
$args = [1, 2, 3];
echo sum(...$args); // 6
Q39 What is a Generator in PHP? ▼

Generators are functions that use yield to produce a sequence of values lazily — one at a time, without building the whole array in memory. Ideal for large datasets.

function csvReader(string $file) {
  $handle = fopen($file, 'r');
  while (($line = fgetcsv($handle)) !== false) {
    yield $line; // produce one row at a time
  }
  fclose($handle);
}

foreach (csvReader('big.csv') as $row) {
  // Process row without loading entire file
}
Q40 What is array_walk vs array_map? ▼
  • array_map: Returns a NEW array. Does not modify original. Cannot access key by default.
  • array_walk: Modifies array IN PLACE by reference. Callback receives value and key. Returns bool.
$prices = ['apple' => 1.5, 'banana' => 0.5];

// array_walk — modify in place with key access
array_walk($prices, function(&$price, $fruit) {
  $price = "$fruit: \$$price";
});
// ['apple' => 'apple: $1.5', ...]
Q41 What is type hinting and return type declarations in PHP? ▼

Type hinting enforces that function arguments and return values must be of a specific type. Introduced progressively from PHP 5 (class types) through PHP 8.x (union types, intersection types, never).

function divide(float $a, float $b): float {
  if ($b === 0.0) throw new DivisionByZeroError();
  return $a / $b;
}

// Union types (PHP 8.0)
function parse(int|string $id): array|null { ... }

// Nullable type
function find(int $id): ?User { ... } // User or null

// Never (PHP 8.1) — function always throws/exits
function fail(string $msg): never { throw new Exception($msg); }
Q42 What are PHP's date and time functions? ▼ 
echo date('Y-m-d H:i:s');         // 2025-03-15 14:30:00
echo time();                       // Unix timestamp
echo strtotime('next Monday');   // timestamp
echo date('D', strtotime('+7 days')); // day name

// Modern: DateTime class (preferred)
$dt = new DateTime('now', new DateTimeZone('Asia/Kolkata'));
echo $dt->format('d M Y');        // 15 Mar 2025
$dt->modify('+30 days');
$diff = $dt->diff(new DateTime());
echo $diff->days; // days between
Q43 What is a recursive function? Give an example. ▼

A recursive function calls itself to solve sub-problems. Every recursive function must have a base case (exit condition) to prevent infinite loops.

function factorial(int $n): int {
  if ($n <= 1) return 1;      // base case
  return $n * factorial($n - 1); // recursive case
}
echo factorial(5); // 120

// Useful for directory traversal
function listFiles(string $dir): array {
  $result = [];
  foreach (glob("$dir/*") as $path) {
    $result[] = $path;
    if (is_dir($path)) {
      $result = array_merge($result, listFiles($path));
    }
  }
  return $result;
}
Q44 What is the difference between isset(), empty(), and is_null()? ▼
  • isset($var): Returns true if variable exists AND is not null. Safe to use on undefined vars.
  • empty($var): Returns true if variable is: "", 0, "0", null, false, [], or undefined.
  • is_null($var): Returns true only if value is null. Throws notice if var is undefined.
$a = "";   $b = 0;   $c = null;

isset($a);    // true  (exists, not null)
empty($a);    // true  (empty string)
is_null($a);  // false (not null)

isset($c);    // false (null)
empty($c);    // true
is_null($c);  // true

round(4.555, 2);   // 4.56
max(1, 3, 2);      // 3

🟠 Database — Q46 to Q55

MySQL, PDO, MySQLi, and database best practices with PHP

Q46 What is the difference between MySQLi and PDO? ▼
  • MySQLi: Works only with MySQL. Has both OOP and procedural API. Supports multiple statements.
  • PDO (PHP Data Objects): Database-agnostic — works with MySQL, PostgreSQL, SQLite, Oracle, etc. Only OOP API. Better for portability.
Use PDO for new projects — switching databases later only requires changing the DSN string.
Q47 How do you connect to a MySQL database using PDO? ▼ 
try {
  $pdo = new PDO(
    'mysql:host=localhost;dbname=mydb;charset=utf8mb4',
    'username',
    'password',
    [
      PDO::ATTR_ERRMODE            => PDO::ERRMODE_EXCEPTION,
      PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC,
      PDO::ATTR_EMULATE_PREPARES   => false,
    ]
  );
} catch (PDOException $e) {
  error_log($e->getMessage());
  die('DB connection failed');
}
Q48 What are prepared statements and why are they important? ▼

Prepared statements separate SQL code from data, preventing SQL injection attacks. The query structure is sent first, then the data is bound separately — the database never mixes them up.

// ❌ VULNERABLE — SQL injection possible
$sql = "SELECT * FROM users WHERE email = '$email'";

// ✅ SAFE — prepared statement
$stmt = $pdo->prepare("SELECT * FROM users WHERE email = ?");
$stmt->execute([$email]);
$user = $stmt->fetch();

// Named placeholders
$stmt = $pdo->prepare("INSERT INTO users (name, email) VALUES (:name, :email)");
$stmt->execute([':name' => $name, ':email' => $email]);
Q49 How do you perform CRUD operations with PDO? ▼ 
// CREATE
$stmt = $pdo->prepare("INSERT INTO products (name, price) VALUES (?, ?)");
$stmt->execute(['Widget', 9.99]);
$id = $pdo->lastInsertId();

// READ
$stmt = $pdo->prepare("SELECT * FROM products WHERE id = ?");
$stmt->execute([$id]);
$product = $stmt->fetch();          // one row
$all     = $stmt->fetchAll();       // all rows

// UPDATE
$pdo->prepare("UPDATE products SET price = ? WHERE id = ?")
    ->execute([12.99, $id]);

// DELETE
$pdo->prepare("DELETE FROM products WHERE id = ?")
    ->execute([$id]);
Q50 How do you handle transactions in PDO? ▼

Transactions group multiple queries so they either all succeed (commit) or all fail (rollBack) — ensuring data integrity.

try {
  $pdo->beginTransaction();

  $pdo->prepare("UPDATE accounts SET balance = balance - ? WHERE id = ?")
      ->execute([500, $senderId]);

  $pdo->prepare("UPDATE accounts SET balance = balance + ? WHERE id = ?")
      ->execute([500, $receiverId]);

  $pdo->commit(); // Both succeed
} catch (Exception $e) {
  $pdo->rollBack(); // Undo both on failure
  throw $e;
}
Q51 What is the difference between fetch() and fetchAll()? ▼
  • fetch(): Returns the next single row from the result set. Memory-efficient for large results — iterate row by row.
  • fetchAll(): Returns ALL rows into a PHP array at once. Convenient but uses more memory for large datasets.
// fetchAll — good for small results
$users = $stmt->fetchAll(PDO::FETCH_ASSOC);

// fetch in loop — good for large results
while ($row = $stmt->fetch(PDO::FETCH_ASSOC)) {
  processRow($row);
}

// Fetch as object
$stmt->fetchAll(PDO::FETCH_OBJ);
$stmt->fetchAll(PDO::FETCH_CLASS, 'User');
Q52 What is SQL Injection and how does PHP prevent it? ▼

SQL Injection is an attack where a malicious user inserts SQL code into input fields to manipulate database queries.

// Attack: user enters: ' OR '1'='1
// This query returns ALL users!
"SELECT * FROM users WHERE password = '' OR '1'='1'"

// Prevention 1: Prepared statements (best)
$stmt = $pdo->prepare("SELECT * FROM users WHERE password = ?");
$stmt->execute([$_POST['password']]);

// Prevention 2: quote() (less preferred)
$safe = $pdo->quote($_POST['input']);
Q53 How does PHP handle file uploads? ▼ 
// HTML form
// <form method="POST" enctype="multipart/form-data">
// <input type="file" name="photo">

// PHP handler
if ($_FILES['photo']['error'] === UPLOAD_ERR_OK) {
  $allowed = ['image/jpeg', 'image/png'];
  $type    = mime_content_type($_FILES['photo']['tmp_name']);

  if (!in_array($type, $allowed)) die('Invalid file type');
  if ($_FILES['photo']['size'] > 2_000_000) die('Too large');

  $dest = 'uploads/' . uniqid() . '.jpg';
  move_uploaded_file($_FILES['photo']['tmp_name'], $dest);
}
Never trust the user-supplied file name or MIME type from $_FILES. Always validate with server-side checks.
Q54 What is the difference between mysql_real_escape_string and prepared statements? ▼
  • mysql_real_escape_string: Escapes special characters in a string for use in SQL. Deprecated in PHP 5.5, removed in PHP 7. Error-prone — easy to forget to apply it.
  • Prepared Statements: Structurally separate SQL from data. No chance of forgetting. Works across all databases via PDO. The correct, modern approach.
Never use the old mysql_* functions. They are removed from PHP. Use PDO or MySQLi with prepared statements.
Q55 What is PHP's ORM and what is the difference between active record and data mapper? ▼

An ORM (Object-Relational Mapper) maps database rows to PHP objects, abstracting raw SQL queries.

  • Active Record: Model class directly interacts with the database. Simple but tightly coupled. Used by Laravel's Eloquent. User::find(1), $user->save().
  • Data Mapper: Model is a plain object (no DB logic). A separate Mapper/Repository handles persistence. Looser coupling, better testability. Used by Doctrine ORM.

🔴 Security — Q56 to Q63

Common vulnerabilities and how to prevent them in PHP applications

Q56 What is XSS and how do you prevent it in PHP? ▼

XSS (Cross-Site Scripting) is an attack where malicious scripts are injected into web pages viewed by other users.

// Attack: user submits: <script>steal(document.cookie)</script>

// Prevention: always escape output
echo htmlspecialchars($userInput, ENT_QUOTES, 'UTF-8');

// Or htmlentities for full encoding
echo htmlentities($userInput, ENT_QUOTES, 'UTF-8');

// Also set Content-Security-Policy header
header("Content-Security-Policy: default-src 'self'");
Rule: Escape on output, not on input. Sanitize input for storage; escape for the output context (HTML, JS, URL, CSS).
Q57 What is CSRF and how do you protect against it? ▼

CSRF (Cross-Site Request Forgery) tricks a logged-in user's browser into making an unwanted request to your server using their active session.

// Generate token on form render
session_start();
if (empty($_SESSION['csrf_token'])) {
  $_SESSION['csrf_token'] = bin2hex(random_bytes(32));
}

// Embed in form
// <input type="hidden" name="csrf" value="<?= $_SESSION['csrf_token'] ?>">

// Validate on form submission
if (!hash_equals($_SESSION['csrf_token'], $_POST['csrf'] ?? '')) {
  die('CSRF validation failed');
}
Q58 How do you securely hash and verify passwords in PHP? ▼

Use PHP's built-in password_hash() and password_verify() functions. They use bcrypt (or Argon2 in PHP 7.2+) with automatic salting.

// Registration — hash password before storing
$hash = password_hash($_POST['password'], PASSWORD_BCRYPT);
// Store $hash in database

// Login — verify against stored hash
$stored = $user['password_hash']; // from DB
if (password_verify($_POST['password'], $stored)) {
  // Login successful
}

// Re-hash if algorithm needs upgrade
if (password_needs_rehash($stored, PASSWORD_BCRYPT)) {
  $newHash = password_hash($password, PASSWORD_BCRYPT);
  // Update in DB
}
Never use md5() or sha1() for passwords — they are not designed for password hashing and are easily cracked.
Q59 What is session fixation and session hijacking? How do you prevent them? ▼
  • Session fixation: Attacker sets a known session ID before user logs in, then uses it to hijack the session.
  • Session hijacking: Attacker steals an active session ID (via XSS, network sniffing) and impersonates the user.
session_start();

// Regenerate ID on login (prevents fixation)
session_regenerate_id(true);
$_SESSION['user_id'] = $userId;

// Secure session cookie settings
session_set_cookie_params([
  'httponly' => true,   // no JS access
  'secure'   => true,   // HTTPS only
  'samesite' => 'Strict'
]);
Q60 What is input validation vs sanitization in PHP? ▼
  • Validation: Checking if the input meets requirements (email format, number range, required field). Does not modify data.
  • Sanitization: Cleaning the input by removing or encoding unwanted characters. Modifies data.
// Validation using filter_var
$email = $_POST['email'];
if (!filter_var($email, FILTER_VALIDATE_EMAIL)) {
  die('Invalid email');
}

// Sanitization using filter_var
$clean = filter_var($_POST['name'], FILTER_SANITIZE_SPECIAL_CHARS);
$url   = filter_var($_POST['url'],  FILTER_SANITIZE_URL);
$num   = filter_var($_POST['age'],  FILTER_VALIDATE_INT);
Q61 What are PHP security best practices? ▼
  • Keep PHP and all libraries updated
  • Use prepared statements for all DB queries
  • Use password_hash() for passwords — never md5/sha1
  • Escape all output with htmlspecialchars()
  • Implement CSRF tokens on all state-changing forms
  • Set display_errors = Off in production, log to file
  • Use HTTPS everywhere; set Secure, HttpOnly, SameSite cookie flags
  • Validate file uploads: check MIME type server-side, restrict extensions
  • Use open_basedir to restrict file access in php.ini
  • Rate-limit login endpoints to prevent brute force
  • Use Content Security Policy (CSP) headers
Q62 What is the difference between md5(), sha1(), and password_hash()? ▼
  • md5() / sha1(): Cryptographic hash functions designed for speed — great for checksums, not passwords. No salt, fast to brute-force with GPUs.
  • password_hash(): Specifically designed for passwords. Adds random salt automatically. Uses slow algorithms (bcrypt/Argon2) so brute-force is computationally expensive.
echo md5("secret");     // 5ebe2294ecd0e0f... (fast, NO salt)
echo password_hash("secret", PASSWORD_BCRYPT);
// $2y$10$randomsalt$hashedvalue... (slow, auto-salted)
Q63 How do you prevent directory traversal attacks? ▼

Directory traversal attacks use ../ sequences to access files outside the intended directory.

// Attack: ?file=../../etc/passwd
$file = $_GET['file']; // ../../etc/passwd
readfile("/var/www/files/" . $file); // DANGEROUS!

// Prevention: use realpath and check prefix
$basePath = realpath('/var/www/files');
$filePath = realpath($basePath . '/' . $_GET['file']);

if ($filePath === false || strpos($filePath, $basePath) !== 0) {
  die('Access denied');
}
readfile($filePath); // safe

🟣 Advanced PHP — Q64 to Q72

Performance, design patterns, modern PHP features, and architecture

Q64 What is PHP Composer and autoloading? ▼

Composer is PHP's dependency manager. It downloads and manages packages, and generates an autoloader so you can use any class without manual require calls.

# Install a package
composer require guzzlehttp/guzzle

# composer.json defines dependencies
{
  "require": { "monolog/monolog": "^3.0" },
  "autoload": { "psr-4": { "App\\": "src/" } }
}

# In your PHP entry point
require_once 'vendor/autoload.php';

# Now all classes autoload — no manual require needed
use App\Models\User;
$user = new User();
PSR-4 is the modern autoloading standard — namespace maps to directory structure.
Q65 What are PHP Design Patterns? Name and explain common ones. ▼
  • Singleton: One instance only (DB connection). See Q23.
  • Factory: Creates objects without specifying exact class — hides creation logic.
  • Observer: One object (Subject) notifies multiple dependent objects (Observers) when state changes. Used in event systems.
  • Strategy: Define a family of algorithms, encapsulate each, make them interchangeable (e.g., different payment methods).
  • Repository: Abstracts data storage — controllers don't know if data comes from MySQL, Redis, or a file.
  • Decorator: Adds behavior to objects dynamically without modifying their class.
Q66 What are PHP Fibers? (PHP 8.1+) ▼

Fibers are PHP 8.1's lightweight concurrency primitives — like coroutines. A Fiber can suspend its own execution (suspend()) and be resumed later, enabling cooperative multitasking.

$fiber = new Fiber(function(): void {
  echo "Fiber started\n";
  $value = Fiber::suspend('hello'); // suspend, return 'hello'
  echo "Fiber resumed with: $value\n";
});

$returned = $fiber->start();       // "Fiber started", $returned = 'hello'
$fiber->resume('world');           // "Fiber resumed with: world"
Fibers power async PHP libraries like ReactPHP and Amp — enabling high-concurrency without threads.
Q67 What is PHP-FPM and how does it differ from Apache mod_php? ▼
  • mod_php: PHP runs as an Apache module inside each Apache process. Simple setup, but every Apache process loads PHP even for static files. Memory-heavy.
  • PHP-FPM (FastCGI Process Manager): PHP runs as a separate pool of processes. Nginx/Apache communicate via FastCGI. More efficient — PHP workers only handle PHP requests. Supports process pool tuning, logging, and slow request detection.
PHP-FPM + Nginx is the modern standard for production PHP deployments.
Q68 What is OPcache and how does it improve PHP performance? ▼

PHP is interpreted — each request normally parses and compiles PHP files to opcodes. OPcache stores the compiled bytecode in shared memory, skipping the parse/compile step on subsequent requests — dramatically improving performance (often 2–10x).

; php.ini — enable OPcache
opcache.enable=1
opcache.memory_consumption=128
opcache.max_accelerated_files=10000
opcache.revalidate_freq=60    ; check for changes every 60s
opcache.validate_timestamps=0 ; disable in production for max speed
Always enable OPcache in production. Tools like Preloading (PHP 7.4+) can load framework files into memory at startup for even more speed.
Q69 What is PSR (PHP Standards Recommendations)? ▼

PSRs are coding standards published by the PHP Framework Interoperability Group (PHP-FIG) to ensure interoperability between PHP packages and frameworks.

  • PSR-1: Basic coding standards (class names, constants)
  • PSR-2 / PSR-12: Extended coding style guide (indentation, braces)
  • PSR-4: Autoloading standard (namespace → directory mapping)
  • PSR-7: HTTP Message interface (Request/Response)
  • PSR-11: Container interface (dependency injection containers)
  • PSR-14: Event dispatcher interface
  • PSR-15: HTTP server middleware interface
  • PSR-17: HTTP factories
Q70 What are PHP Attributes? (PHP 8.0+) ▼

Attributes are structured metadata added to classes, methods, properties, or parameters using #[...] syntax. They replace docblock annotations (like in Doctrine) with native, parseable metadata.

#[Attribute]
class Route {
  public function __construct(public string $path) {}
}

class UserController {
  #[Route('/users')]
  public function index() { ... }
}

// Read via Reflection API
$ref   = new ReflectionMethod(UserController::class, 'index');
$attrs = $ref->getAttributes(Route::class);
echo $attrs[0]->newInstance()->path; // '/users'
Q71 What is PHP's match expression? How is it different from switch? ▼
  • match (PHP 8.0): Strict comparison (===). Returns a value. No fall-through. Throws UnhandledMatchError if no arm matches. Shorter syntax.
  • switch: Loose comparison (==). Falls through by default (needs break). Doesn't return a value directly. Older, more verbose.
$status = 2;

// match — strict, concise, returns value
$label = match($status) {
  1       => 'Active',
  2, 3    => 'Pending',  // multiple conditions
  default => 'Unknown',
};
echo $label; // Pending
Q72 What are the major new features in PHP 8.x? ▼

PHP 8.0 (2020):

  • JIT (Just-In-Time) Compiler — significant performance boost for CPU-bound code
  • Union types: int|string
  • Named arguments: array_slice(array: $a, offset: 1)
  • Match expression (see Q71)
  • Nullsafe operator: $user?->address?->city
  • Attributes (see Q70)
  • Constructor property promotion: __construct(public string $name)

PHP 8.1 (2021): Enums, Fibers, Readonly properties, Intersection types, never return type

PHP 8.2 (2022): Readonly classes, Disjunctive Normal Form types, true/false/null as standalone types

PHP 8.3 (2023): Typed class constants, json_validate(), array_find(), readonly property improvements
